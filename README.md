공부한 네트워크 핵심 이론과 실습 내용을 기록하는 공간입니다.

------

## 🌐 네트워크 구성도 살펴보기
### 🏠 1. 홈 네트워크
* **구성 흐름 :** `인터넷` ➡️ `공유기` ➡️ `PC` 또는 `스마트폰`
> 💡 **공유기의 역할:** 인터넷(외부 네트워크)과 집 안의 기기들(내부 네트워크)을 서로 연결하고 데이터를 중중개해 주는 핵심 장비입니다.

---

### 🏢 2. 데이터 센터 네트워크 (3티어 아키텍처)
* **구성 흐름 :** `인터넷` ➡️ `방화벽` ➡️ `로드밸런서` ➡️ `WEB 서버` ➡️ `WAS 서버` ➡️ `DB 서버`
* **3티어 아키텍처란?**
  * 웹 서버(WEB), 애플리케이션 서버(WAS), 데이터베이스(DB)를 각각 독립된 계층으로 쪼갠 구조입니다.
  * **장점 :** 서버가 분리되어 있어 **보안**성이 높고, 트래픽 분산(**성능**) 및 관리(**유지보수**) 측면에서 매우 유리합니다.


## 🌐 크기에 따른 네트워크 분류 : LAN 과 WAN
* **LAN이란?** Local Area Network, 건물 안이나 특정 지역을 범위로 하는 네트워크.
* **WAN이란?** Wide Area Network, ISP가 제공하는 서비스를 사용하여 구축한 네트워크.

## 🌐 연결 형태에 따른 분류 : 네트워크 토폴로지
* 네트워크는 연결 형태에 따라 `스타형(성형)` 과 `메시형(망형)`으로 나눌 수 있다.
* 스타형 : 중앙 장비에 모든 장비가 연결된 형태, 장점 : 장애 찾기 쉽다, 관리가 쉽다. 단점 : 중앙 스위치 죽으면 전체 영향이간다. 
* 메시형 : 여러 노드들이 서로 전부 연결된 형태이며 일부 장비가 고장나도 다른 경로를 통해 통신이 가능 WAN에서 주로 사용되는 방식, 안전성 높지만 비용이 높다
* 버스형 : 하나의 선 공유 과거에 주요사용됨
* 링형 : 원형 연결, 일부 특수 환경에서 사용한다.
* 대부분의 네트워크는 LAN과 WAN이 혼합된 형태이다. **

## 🌐 네트워크 통신 방식 : 유니캐스트, 멀티캐스트, 브로드캐스트
* 유니캐스트 : 특정 대상과만 통신하는 방식 ex) 특정 사람과 통신, 1대1 통신, 카카오톡, SSH, 웹서핑 등
* 멀티캐스트 : 특정한 다수와 통신하는 방식 ex) 특정 그룹과 통신, IPTV
* 브로드캐스트 : 같은 네트워크 대역에 있는 모든 사용자와 통신하는 방식 ex) 모든 사람과 통신, ARP 요청
* 애니캐스트 : 가장 가까운 서버로 전달 ex) DNS 서버

## 🌐 프로토콜 : 네트워크 통신의 약속
* 프로토콜 : 컴퓨터가 네트워크 망에서 특정 컴퓨터와 통신할 때는 프로토콜이라는 것을 사용한다. 프로토콜은 일종의 약속 또는 양식이다.
* 프로토콜에는 이더넷, ICMP, IP, TCP, UDP 등 여러 종류가 있으며, 각각 사용하는 네트워크 주소가 다르다.
* IP : 주소지정
* TCP : 신뢰성
* UDP : 속도
* HTTP : 웹 통신
* HTTPS : 암호화 웹
* DNS : 이름 -> IP 변환
* ARP : IP -> MAC 변환
* ICMP : 네트워크 상태 확인
* SSH : 원격접속
* 흐름 : google.com 입력 -> DNS가 IP 찾음 -> TCP 연결 -> HTTP 요청 -> 서버가 응답하는 흐름

## 🌐 패킷의 구조와 역할
* 헤더, 페이로드, 트레일러 구조
* 패킷은 네트워크를 통해 주고받는 메시지의 기본 단위
* 헤더는 대부분 포함되지만, 트레일러는 없을 수도 있다.
* 헤더는 패킷의 송수신지 정보를 담고 있다.
* 1계층 bit
* 2계층 프레임
* 3계층 패킷
* 4계층 세그먼트

## 🌐 TCP/IP 모델 과 OSI 7 레이어 모델
* TCIP/IP 모델 : 네트워크 엑세스, 인터넷, 전송, 응용 4개 계층으로 이루어져있다.
* OSI 7레이어 모델은 ISO에서 지정한 모델로, 통신 전체를 체계적으로 다룬다.
* OSI 7계층 : 응용계층, 표현계층, 세션계층, 전송계층, 네트워크계층, 데이터링크계층, 물리계층 (응표세전네데물)
  * 1계층 물리계층 : 전기 신호 전달, 케이블, 허브
  * 2계층 데이터링크 : MAC 주소 기반 통신, 스위치, 같은 네트워크안에서 통신
  * 3계층 네트워크 : IP 주소 기반, 라우터, 다른 네트워크 찾기
  * 4계층 전송 : TCP/UDP 담당, PDU:세그먼트, 신뢰성/속도 관리
  * 5계층 세션 : 연결 관리
  * 6계층 표현 : 암호화/압축, SSL/TLS
  * 7계층 응용 : 사용자 프로그램, 브라우저, 메신저 / HTTP, FTP, SMTP
* 이더넷, IP, ICMP, ARP, TCP, UDP, HTTP 등의 프로토콜이 있다.
* TCP/IP가 더 실무적으로 되어있다.

## 🌐 인캡슐레이션/디캡슐레이션
* 패킷은 페이로드(데이터), 헤더(제어정보), 푸터(꼬리말) 로 구성되는데 대부분의 경우 헤더만 사용된다.
* 페이로드에 프로토콜을 헤더로 붙이는 과정을 캡슐화(Encapsulation)이라고 한다.
* 캡슐화는 상위 계층에서 하위 계층으로 내려가면서 프로토콜을 붙이는 과정이다.
* 패킷을 받는 쪽에서 디캡슐화(Decapsulation)과정을 통해 패킷을 확인한다.
* 하위 프로토콜부터 하나씩 벗겨내며 데이터를 확인하는 과정으로, 누가 누구에게 어떤 데이터를 보냈는지 순차적으로 파악한다
* 하위계층에서부터 상위 계층으로 올라가며 프로토콜 헤더를 제거하고 최종적으로 순수한 데이터만 얻게 된다.
* 인캡슐레이션 (응용데이터 -> TCP Header(포트 번호 포함)  -> IP Header(IP주소포함) -> Ethernet Header(MAC주소포함) -> 전송)
* 디캡슐레이션 (Ethernet제거 -> IP 제거 -> TCP 제거 -> 응용데이터 도착

## 🌐 네트워크 연결 부분
* 네트워크는 유선, 무선 연결방식으로 나뉨
* 유선 네트워크 대표는 이더넷(Ethernet)
  * `이더넷 Ethernet`
     * 현재 LAN의 표준 기술
     * MAC 주소를 사용
     * 스위치 기반
     * 프레임 단위 통신
     * CSMA/CD 기반(과거 허브 환경) : 먼저 통신 중인지 확인 하고 충돌 나면 기다렸다 재전송 요즘은 스위치 사용으로 충돌 거의 없음
* 반이중 /전이중
   * 반이중(Half Duplex) 통신 : 한 번에 한 방향만 통신한다. ex) 무전기
   * 전이중(Full Duplex) 통신 : 동시에 송수신 가능하다. ex) 전화

 ## 🌐 네트워크 회선
 * 회선이란? 데이터가 이동하는 통로 ex) 인터넷 회선, 광케이블, 전용선
 * 회선의 종류
    * 전용 회선 : 기업에서 많이 사용
    * 인터넷 회선 : 일반 인터넷, 여러 사용자가 공유
    * 인터넷 전용 회선 : 인터넷은 쓰는데 품질을 보장받는 회선, 기업에서 주로 사용
    * VPN : Virtual Private Network 인터넷 위에 안전한 전용망처럼 사용하는 기술
    * 광케이블 : 빛으로 데이터를 전달, 빠르고, 노이즈적고, 장거리 가능하다.
  
 ## 🌐 네트워크 구성 요소
 * NIC : Network Interface Card 컴퓨터를 네트워크에 연결하는 역할
 * 케이블과 커넥터 : CAT5e (1G), CAT6(10G 일부), CAT6A(10G)
 * 허브 : 1계층 장비, 들어온 데이터를 모든 포트로 전달한다. 충돌 많고, 보안에 취약
 * 스위치 : 2계층 장비, MAC 주소 기반 전달한다. MAC Address Table이 있다
 * 라우터 : 3계층 장비, 다른 네트워크와 연결, 라우팅 테이블
 * 로드밸런서 : 트래픽 분산 장비 : 서버 여러 대로 분산, 부하 분산, 장애 대응, 성능 향상
 * 보안장비 : 방화벽 (허용과차단), IPS(이상 트래픽 탐지/차단)
 * IDS/ IPS : IDS 침입탐지, 알림중심 / IPS : 침입차단, 자동 대능 가능
 * VPN : 가상 사설망, 인터넷을 통해 안전하게 내부망 접속
 * NAT : 사설IP <-> 공인 IP 변환
 * DHCP : IP 자동 할당

  ## 🌐 MAC 주소
  * 랜카드(NIC)에 부여된 물리 주소
  * 2계층 주소
  * OO:1A:2B:3C:4D:5E / 48비트 사용
  * 전 세계 유일하며, 제조사 정보 포함되어있고, 변경 가능은 하지만 기본은 고유값이다.
  * MAC 주소는 앞 24비트는 제조사 뒤 24비트는 장비 고윳값이다.

 ## 🌐 IP 주소
  * 네트워크에서 사용하는 논리 주소
  * IPv4 형태, 192.168.0.1 32비트
  * 다른 네트워크 찾기 위해 필요하다.
  * 라우터가 IP 기반 동작한다.
  * 공인IP, 사설IP
     * 공인IP : 인터넷에서 유일한 주소(통신사 제공)
     * 사설IP : 내부망용 주소 / 192.168.x.x , 10.x.x.x, 172.16.x.x
     * 사설 IP는 IPv4 부족때문에 사용한다.
  * NAT : 사설 IP <-> 공인 IP 변환
  * 서브넷 마스크 : 네트워크/호스트 구분용도
  * CIDR 표기 : 앞 24비트가 네트워크 부분이다.
  * 기본 게이트웨이 : 외부로 나갈 출구
  * DNS : 도메인 -> IP 변환 ex) goolgle.com -> 142.x.x.x
  * IP 통신 과정 : 내 pc -> 게이트웨이 -> 라우터 -> 인터넷 -> 목적지 서버
  * TTL : Time To Live , 패킷 생존 시간, 라우터 지날 때 감소하며 0 되면 폐기, 루프 방지 목적이다.

 ## 🌐 TCP와 UDP
  * TCP 는 신뢰성 중심, 연결형, 순서보장, 재전송, 흐름제어 , 혼잡제어
  * TCPsms 3-Way Handshaek 로 연결
     * SYN, SYN-ACK, ACK
  * TCP는 안정적이고 데이터 손실 복구가 되나 느리다. 주로 웹, 로그인, 파일전송 , DB에서 사용한다.
  * UDP는 속도 중심이고 비연결형, 빠르다, 신뢰성없다, 순서보장 하지 않는다.
  * 주로 게임, 스트리밍 에서 사용한다.

  ## 🌐 이더넷의 이해와 중요성
   * 이더넷은 현대 LAN 환경, 특히 유선 LAN에서 가장 널리 사용되는 기술이다.
   * 물리 계층에서의 통신 매체 간 정보 교환 규칙과 데이터 링크 계층에서 두 호스트 간 메시지 교환 형식을 정의하는 핵심 기술이다.
   * 오늘날 대부분의 유선 LAN 환경은 이더넷 기술을 기반으로 구성된다.
   * 물리 계층의 케이블과 데이터 링크 계층에서 주고받는 프레임이 모두 이더넷 규격을 따른다.

  ## 🌐 이더넷 프레임
   * 데이터 링크 계층에서 메시지를 주고받는 단위는 `프레임`이라고 한다.
   * 이더넷 네트워크에서 사용되는 프레임을 이더넷 프레임 이라 한다.
   * 이더넷 프레임은 헤더, 페이로드(데이터), 트레일러로 구성된다.
   * 이더넷 프레임의 헤더는 크게 프리앰블, 수신지 MAC 주소(6바이트), 송신지 MAC 주소(6바이트), 타입(2바이트), 길이 필드로 구성된다.
   * 프리앰블은 8바이트 길의 비트열로, 수신지에서 프레임 수신을 준비하도록 동기화하는 역할을 한다.
   * MAC 주소는 수신지 및 송신지 MAC 주소 필드에 명시되는 네트워크 인터페이스마다 부여되는 6바이트(48비트) 길이의 물리적 주소이다.

  ## 🌐 NIC(네트워크 인터페이스 컨트롤러)란?
   * NIC는 호스트와 통신 매체를 연결하는 인터페이스 역할을 수행하는 장비이며 흔히 네트워크 카드, 랜 카드 등으로 불린다.
   * NIC는 통신 매체의 전기신호를 호스트가 이해할 수 있는 정보로 변환하거나, 호스트의 디지털 정보를 통신 매체 신호로 변환하는 역할을 한다.
   * 호스트가 네트워크를 통해 송수신하는 모든 정보는 NIC를 거치게 된다.
   * NIC는 MAC 주소를 인식하여 수신한 프레임의 송수신지 MAC 주소를 확인하고, 자신과 관련 없는 프레임을 폐기하거나 일치하는 프레임을 역캡슐화 수행한다.

  ## 🌐 물리 계층과 데이터 링크 계층 장비의 핵심 차이
   * 물리 계층의 네트워크 장비는 주소 개념이 없어 송수신되는 정보에 대한 조작이나 판단을 할 수 없다.
   * 데이터 링크 계층 이상의 장비는 MAC 주소와 같은 주소 개념을 가지고 있어 송수신 정보를 특정하고 판단하여 조작할 수 있다.
  
  ## 🌐 물리계층 대표 장비 `허브` (1계층)
   * 허브는 `리피터 허브` 또는 `이더넷 허브` 등으로 불리며, 여러 호스트를 연결하는데 사용된다.
   * 허브의 첫 번째 전송 특징은 모든 포트로 해당 메시지를 내보낸다. 이러한 것은 불필요한 트래픽을 과도하게 증가시키는 문제점을 야기시킨다.
   * 허브의 두 번재 특징은 반이중 방식으로 통신한다. ex) 무전기
   * 모든 포트로 신호를 보내고 반이중 방식으로 통신하는 특성 때문에 충돌(Collision)문제가 발생한다. 충돌이 발생할 수 있는 영역을 콜리전 도메인 이라고 한다.
   * 충돌을 줄이려고 CSMA/CD 프로토콜을 이용한다.

  ## 🌐 데이터링크계층 대표 장비 `스위치` (2계층)
   * 스위치는 허브와 달리 어떤 MAC 주소를 가진 호스트가 어떤 포트에 연결되어 있는지 식별할 수 있다.
   * 스위치는 전 이중 모드 통신을 지원하여 근본적으로 CSMA/CD 프로토콜이 필요가 없다.
   * 일반적으로 스위치라하면 L2 스위치를 말하긴한다.
   
   * 스위치의 주요기능으로는 첫 번째는 `MAC 주소 학습 기능`이다.
      * 이 기능은 어떤 포트에 어떤 MAC 주소를 가진 호스트가 연결되어 있는지 학습하여 기억한다.
      * 스위치가 전달받은 신호를 원하는 목적지 MAC 주소가 연결된 포트로만 내보낼 수 있게 해준다.
      * MAC 주소 학습을 위해 스위치는 MAC 주소 테이블이라는 특별한 정보를 저장한다.
      * 이 테이블은 스위치 포트와 연결된 호스트의 MAC 주소 간의 연관 관계를 나타내는 표와 같으며, 스위치는 이 테이블을 통해 특정 포트에 연결된 호스트의 MAC 주소를 알 수 있다.
      * MAC 주소 테이블은 플러딩, 포워딩, 필터링, 에이징 을 통해 생성되고 관리된다.
      * `스위치 플러딩`은 처음에 MAC 주소 테이블이 비어있어 호스트의 위치를 모르면 허브처럼 모든 포트로 프레임을 전송한다.
      * MAC 주소를 학습한 이후에는 A->C로 전송할 때 다른 포트로 보내지 않는다 이것을 `필터링`이라고한다.
      * 연결된 호스트에 포트로만 프레임을 내보내는것을 `포워딩`이라고 한다.
      * `에이징`은 MAC 주소 테이블에 있는 항목을 일정 시간이 지나면 삭제하는 기능이다.
    
   * 스위치의 두 번째 주요 기능은 `VLAN(Virtual LAN)` 이다.
      * VLAN은 논리적으로 나누어진 가상의 LAN을 의미하며, 한 대의 스위치로 여러 개의 가상 LAN을 만들 수 있게 해준다.
      * VLAN을 이용하면 불필요한 트래픽을 분배하여 성능 저하를 방지할 수 있다.
      * VLAN을 사용하면 물리적인 스위치가 한 대라도 여러 대의 스위치가 있는 것처럼 호스트의 물리적 위치와 관계없이 논리적인 단위로 LAN을 구성하고 나눌 수 있다.
    
     ## 🌐 STP (Spanning Tree Protocol)
     
     ### 1. STP의 개념 및 필요성
     * **정의:** 가상이나 물리적인 네트워크 루프(Loop)를 방지하기 위해, 특정 포트를 논리적으로 차단하는 **루프 방지 프로토콜**입니다.
     * **실무 활용:** 네트워크의 **이중화(Redundancy)**를 구현하여 특정 장비나 회선에 장애가 발생해도 서비스가 중단되지 않도록 **네트워크 안정성**을 확보합니다.
     
     ---
     
     ### 🔄 2. STP 기본 동작 개념
     스위치 간의 협상을 통해 다음과 같은 순서로 루프를 제거하고 최적의 경로를 계산합니다.
     
     1. **Root Bridge 선출:** 전체 네트워크의 중심이 될 기준 스위치(중앙 스위치)를 선출합니다.
     2. **최적 경로 계산:** Root Bridge를 기준으로 각 스위치까지의 가장 가깝고 효율적인 경로를 계산합니다.
     3. **차단 포트 결정 (루프 제거):** 최적 경로 외에 루프를 유발할 수 있는 나머지 우회 경로는 포트를 차단(Blocking)하여 트래픽을 막습니다.

---

### 🚀 3. 향상된 STP 종류

실무 환경과 네트워크 규모에 따라 기존 STP의 단점을 보완한 프로토콜을 사용합니다.

| 프로토콜명 | 풀네임 (Full Name) | 주요 특징 및 실무 활용 |
| :--- | :--- | :--- |
| **RSTP** | Rapid Spanning Tree Protocol | • **빠른 복구 속도** (기존 STP의 30~50초 걸리던 수렴 시간을 2~3초 내외로 단축)<br>• 현대 실무 네트워크에서 가장 기본적으로 많이 사용됨 |
| **MST** | Multiple Spanning Tree Protocol | • **여러 개의 VLAN을 그룹으로 묶어** STP 인스턴스를 관리<br>• 스위치의 CPU 자원을 아낄 수 있어 **대규모 환경**에 필수적임 |

## 🌐 네트워크 계층 (3계층) 대표적인 장비 `라우터`

### 1. 네트워크 계층과 IP 프로토콜
* **네트워크 계층의 핵심:** 물리 계층과 데이터 링크 계층의 한계를 극복하고, **IP 주소 지정**과 **라우팅**을 가능하게 하는 네트워크 기능의 핵심입니다.
* **IP 프로토콜 ($IPv4$ vs $IPv6$):** 가장 중추적인 역할을 수행하며, 특별한 언급이 없다면 기본적으로 `IPv4`를 의미합니다.
  * **IPv4:** 32비트 구조 (약 43억 개 주소 제한으로 고갈 문제 발생)
  * **IPv6:** 128비트 구조 (IPv4의 한계를 극복하기 위해 등장, 무한대에 가까운 주소 할당 가능)

---

### 📦 2. IP 프로토콜의 핵심 기능

#### 🔹 주소 지정 (Addressing)
* IP 주소 체계를 통해 호스트가 속한 **네트워크 주소**와 **호스트 주소** 자체를 식별하는 기능입니다.
* IPv4 주소는 **32비트**로 구성되며, 4개의 **옥텟(Octet)**으로 구분되어 각 옥텟은 `0~255`까지의 숫자로 표현됩니다.

#### 🔹 단편화 (Fragmentation)
* 전송하려는 패킷 크기가 **MTU(Maximum Transmission Unit, 최대 전송 단위)**보다 클 때 발생합니다.
* **MTU 사양:** 일반적으로 `1500바이트` 크기를 가집니다.
* **작동 방식:** 큰 데이터를 MTU 이하의 작은 패킷 조각으로 쪼개어 전송하며, 수신 측에서 이 패킷들을 다시 하나로 재조합합니다.

#### 📄 IPv4 패킷 헤더의 주요 필드
| 필드명 | 역할 설명 |
| :--- | :--- |
| **식별자 (Identification)** | 단편화된 패킷들이 원래 하나의 데이터였음을 식별하기 위한 고유 번호 |
| **플래그 (Flags)** | 단편화 금지(DF) 및 추가 단편화 존재 여부(MF)를 나타내는 정보 |
| **단편화 오프셋 (Fragment Offset)** | 쪼개진 패킷 조각이 원래 데이터에서 얼마나 떨어져 있는지의 위치 값 |
| **TTL (Time To Live)** | 패킷의 수명. 라우팅 중 패킷이 네트워크 내에서 무한 루프를 도는 것을 방지 |
| **프로토콜 (Protocol)** | 상위 계층 프로토콜을 명시하여 캡슐화된 데이터의 종류를 안내 |
| **송신지 / 수신지 IP** | 패킷의 출발지 IP 주소와 최종 목적지 IP 주소 |

---

### 🔍 3. ARP (Address Resolution Protocol)

* **필요성:** 네트워크 통신 시 IP 주소는 알고 있지만, 데이터 링크 계층 전송에 필수적인 **MAC 주소를 모르는 상황**을 해결하기 위해 사용됩니다. (동일 네트워크 내에서 작동)

#### 🔄 ARP 프로토콜 동작 3단계
1. **ARP 요청 (Broadcast):** MAC 주소를 알아내기 위해 네트워크 전체에 브로드캐스트 메시지를 전송합니다.
2. **ARP 응답 (Unicast):** 해당 IP를 가진 호스트가 자신의 MAC 주소를 담아 요청 호스트에게 유니캐스트로 답변을 보냅니다.
3. **ARP 테이블 갱신:** 응답을 받은 호스트는 IP와 MAC 주소 매핑 정보를 `ARP 테이블`에 기록하고 유지합니다.

> 💡 **ARP 패킷 구성:** 오퍼레이션 코드(요청: 1, 응답: 2), 송신지/수신지 하드웨어 주소(MAC), 송신지/수신지 프로토콜 주소(IP)

---

### 🔢 4. IP 주소 체계의 발전 (Classful vs Classless)

#### 🅰️ 클래스풀 주소 체계 (Classful Addressing)
네트워크 크기에 따라 사전에 고정된 구분을 사용하는 방식입니다. (D, E 클래스는 특수 목적용)
* **A 클래스:** `0.0.0.0` ~ `127.255.255.255` (첫 비트 `0` / 대규모 네트워크)
* **B 클래스:** `128.0.0.0` ~ `191.255.255.255` (첫 두 비트 `10` / 중규모 네트워크)
* **C 클래스:** `192.0.0.0` ~ `223.255.255.255` (첫 세 비트 `110` / 소규모 네트워크)
* **⚠️ 치명적 한계:** 정해진 크기로만 쪼갤 수 있어 IP 주소의 낭비가 매우 심합니다.

#### 🅱️ 클래스리스 주소 체계 (Classless Addressing)
클래스 구애 없이 네트워크 주소 비트를 유연하게 조정하는 방식입니다. (예: 20비트, 25비트 조정 가능)
* **서브넷 마스크 (Subnet Mask):** 클래스가 없기 때문에 네트워크와 호스트 영역을 구별하기 위해 도입되었습니다. (`1`은 네트워크 영역, `0`은 호스트 영역)
* **CIDR (사이더) 표기법:** 서브넷 마스크의 `1`의 개수를 IP 주소 뒤에 직관적으로 표현하는 방식입니다. (예: `/24`)

#### 🧮 네트워크 주소 계산법
* **네트워크 자체 주소:** IP 주소와 서브넷 마스크를 비트 **AND 연산**한 결과 (호스트 주소가 모두 `0`인 상태)
* **브로드캐스트 주소:** 호스트 주소 공간이 모두 `1`인 상태
* **실제 할당 가능한 호스트 IP 범위:** `네트워크 주소 + 1` ~ `브로드캐스트 주소 - 1`

---

### 🛡️ 5. 공인 IP vs 사설 IP & NAT

* **공인 IP 주소 (Public IP):** 전 세계에서 유일하며 인터넷 망에서 직접 통신 가능한 주소입니다.
* **사설 IP 주소 (Private IP):** 사설 네트워크(외부 미공개 내부망) 내부에서만 유효한 주소입니다. 라우터가 할당하며 다른 사설망과 주소가 중복될 수 있습니다.

#### 🔄 NAT (Network Address Translation) 기술
* **정의:** 사설 IP 주소를 공인 IP 주소로 변환하여 외부 인터넷과 통신할 수 있게 돕는 기술입니다.
* **동작:** NAT 라우터가 내부에서 나가는 패킷의 출발지 IP를 공인 IP로 변환하고, 외부에서 들어오는 패킷의 목적지 IP를 사설 IP로 맵핑하여 돌려줍니다.

---

### ⚙️ 6. IP 주소 할당 방식

#### 📍 정적 할당 (Static IP)
* 사용자가 호스트에 수작업으로 고정 IP를 직접 입력하는 방식입니다.
* **필수 입력 값 4가지:** `부여할 IP 주소`, `서브넷 마스크`, `게이트웨이 주소(외부 네트워크 경로)`, `DNS 주소`

#### ⚡ 동적 할당 (Dynamic IP)
* **DHCP (Dynamic Host Configuration Protocol)** 프로토콜을 기반으로 네트워크 접속 시 자동으로 주소를 할당받는 방식입니다.
* 일반적으로 가정이나 사무실의 공유기, 혹은 라우터가 **DHCP 서버** 역할을 수행하여 IP를 나누어 줍니다.

## 🎛️ 스위치(Switch)와 라우터(Router)의 차이

| 구분 | 스위치 (Switch) | 라우터 (Router) |
| :--- | :--- | :--- |
| **OSI 계층** | 데이터 링크 계층 (**2계층** 장비) | 네트워크 계층 (**3계층** 장비) |
| **주소 기반** | **MAC 주소** 기반 관리 | **IP 주소** 기반 관리 |
| **주요 역할** | **같은 네트워크 안**에서의 통신 연결 | **서로 다른 네트워크** 간의 연결 및 통신 |

> **💡 핵심 요약 (라우팅 vs 스위칭)**
> * **라우팅(Routing):** 데이터가 어디로 갈지 **경로를 결정**하는 행위 (판단)
> * **스위칭(Switching):** 결정된 경로로 데이터를 **실제로 전달**하는 행위 (수행)

## 📍 라우터의 동작 방식과 역할

### 1. 라우터와 라우팅의 정의
* **라우터(Router)란?** 패킷이 목적지까지 안전하고 빠르게 갈 수 있도록 **길을 찾아주는 이정표 장비**입니다.
* **라우팅(Routing)이란?** 출발지 A에서 목적지 B까지 가야 할 때, "목적지 B로 가려면 이쪽 길로 가라"고 경로를 지정해 주는 과정입니다.
* **브로드캐스트 차단:** 라우터는 브로드캐스트 패킷이 다른 네트워크로 넘어가지 않도록 차단하여, 전체 네트워크의 부하(Traffic) 증가를 방지합니다.

### 2. 경로 추적 명령어
목적지까지 가는 경로(거치는 라우터 구간)를 확인하고 싶을 때 사용합니다.
* **Windows:** `tracert [목적지 IP 또는 도메인]`
* **Linux / macOS:** `traceroute [목적지 IP 또는 도메인]`

## 🗺️ 라우팅 테이블(Routing Table)의 역할과 구성

라우팅 테이블은 쉽게 말해 라우터가 가지고 있는 **'네트워크 지도'**입니다. 특정 패킷을 받았을 때 어디로 이동시켜야 하는지에 대한 경로 정보가 담긴 표입니다.
* **생성 방식:** 관리자가 직접 수동으로 입력하는 **정적 라우팅(Static)**과, 프로토콜을 통해 자동으로 생성되는 **동적 라우팅(Dynamic)** 방식이 있습니다.

### 1. 라우팅 테이블의 주요 구성 요소
* **수신지 IP 주소 & 서브넷 마스크:** 패킷의 최종 목적지 네트워크 정보입니다.
* **다음 홉 (Next Hop / 게이트웨이):** 최종 목적지로 가기 위해 다음으로 거쳐야 할 이웃 라우터의 주소입니다.
* **네트워크 인터페이스:** 패킷을 외부로 내보낼 라우터 자체의 물리적/논리적 통로입니다.
* **메트릭 (Metric):** 특정 목적지까지 가는 경로가 여러 개일 때, 어떤 길 조은지 우선순위를 정하는 기준 값(비용)입니다.

### 2. 디폴트 라우트 (Default Route)
* 라우팅 테이블 정보 중 가장 기본이 되는 중요한 경로입니다.
* **`0.0.0.0/0`**으로 표현되며, 라우팅 테이블에 명시되지 않은 **모든 알 수 없는 목적지 패킷**을 처리하기 위한 기본 통로를 의미합니다.
* 일반적으로 집이나 회사 내부 네트워크에서 외부 인터넷 망으로 나가기 위한 라우터나 공유기의 주소가 **디폴트 게이트웨이(Default Gateway)**로 설정됩니다.

## 🗺️ 라우팅 분류: 정적(Static) vs 동적(Dynamic)

라우팅은 라우팅 테이블이 생성되고 유지되는 방식에 따라 크게 두 가지로 분류됩니다.

| 구분 | 정적 라우팅 (Static Routing) | 동적 라우팅 (Dynamic Routing) |
| :--- | :--- | :--- |
| **생성 방식** | 관리자가 **수동**으로 경로 입력 | 라우팅 프로토콜에 의해 **자동** 생성 및 관리 |
| **특징** | IP 주소의 정적 할당 방식과 유사 | IP 주소의 동적 할당(DHCP) 방식과 유사 |
| **설정 방법** | Windows의 `route add` 등 운영체제별 명령어 활용 | 라우터 간 경로 정보를 교환하여 최적 경로 계산 |
| **단점/한계** | 네트워크 변화에 대한 유연성이 떨어지고, **휴먼 에러** 발생 확률이 높음 | 라우터의 자원(CPU, 메모리)을 추가로 소모함 |

## 🏢 자율 시스템 (AS, Autonomous System)의 이해

* **정의:** 동일한 라우팅 정책 아래에서 운영되는 **라우터들의 집단**이자 라우팅 프로토콜의 단위가 되는 라우터 그룹입니다.
* **AS 번호:** 공인 IP 주소처럼 각 AS에는 전 세계적으로 고유한 번호가 할당되어 AS 간 통신의 식별자로 사용됩니다.
* **AS 경계 라우터 (ASBR, AS Boundary Router):** AS 외부(다른 AS)와 통신할 때 관문 역할을 하는 경계 라우터로, AS 간 라우팅을 가능하게 합니다.

## 🚦 라우팅 프로토콜의 분류: IGP와 EGP

동적 라우팅의 핵심인 라우팅 프로토콜은 **자율 시스템(AS)을 기준**으로 내부용과 외부용으로 분류됩니다.

### 1. IGP (Interior Gateway Protocol) - AS 내부 라우팅
하나의 AS 내부에서 최적의 경로를 찾기 위해 사용하는 프로토콜입니다.

* **RIP (Routing Information Protocol)**
  * **방식:** 거리 벡터(Distance Vector) 기반
  * **기준:** **홉 수(Hop Count)**를 기준으로 최적 경로를 선택합니다. (패킷이 경유하는 라우터의 수)
* **OSPF (Open Shortest Path First)**
  * **방식:** 링크 상태(Link State) 기반
  * **기준:** 네트워크의 전체 구성(연결 상태, 대역폭 등)을 그래프로 저장한 뒤, **'최단 경로 알고리즘(Dijkstra)'**을 사용하여 최적 경로를 찾습니다.

### 2. EGP (Exterior Gateway Protocol) - AS 간 라우팅
서로 다른 AS 간에 최적의 경로를 찾고 정보를 교환하기 위해 사용하는 프로토콜입니다.

* **BGP (Border Gateway Protocol)**
  * 대표적인 EGP 프로토콜로, 거대한 인터넷 망을 연결하는 동적 라우팅의 핵심입니다.

## 🌐 BGP (Border Gateway Protocol) 요약

### 1. BGP의 분류 및 피어링 (Peering)
* **기본 정의:** AS(자율 시스템) 간 통신에 사용되는 대표적인 **EGP(외부 게이트웨이 프로토콜)**입니다.
* **BGP의 분류:**
  * **eBGP (external BGP):** 서로 다른 AS 간의 통신을 담당합니다.
  * **iBGP (internal BGP):** 동일한 AS 내부의 라우팅을 담당합니다.
* **피어링 (Peering):** BGP 라우터가 다른 AS의 라우터와 연결을 설정하여 '피어(Peer) 관계'를 맺고 경로 정보를 교환 및 유지하는 과정입니다.

---

### ⚙️ 2. BGP의 복잡한 최적 경로 결정 원인
BGP는 RIP나 OSPF에 비해 최적 경로를 결정하는 과정이 복잡하고 유동적입니다.
* **속성 (Attribute):** 경로에 대한 구체적인 부가 정보입니다.
* **정책 (Policy):** AS 관리 주체가 임의로 부여하는 기준입니다. 특정 AS 선호/차단, 보안, 안정성, 성능 등을 고려하여 관리자가 원하는 대로 라우팅 흐름을 제어할 수 있습니다.

---

### 📌 3. BGP의 3대 핵심 속성 (Attribute)

BGP가 최적의 경로를 찾기 위해 참조하는 대표적인 세 가지 속성입니다.

| 속성명 | 주요 특징 및 역할 |
| :--- | :--- |
| **AS-PATH** | • 메시지가 목적지까지 가면서 **거쳐온 AS 번호들의 목록(경로)**입니다.<br>• 이를 기반으로 경로를 결정하므로 BGP를 **'경로 벡터 라우팅 프로토콜'**이라고 부릅니다. (RIP의 홉 수와 대비됨) |
| **NEXT_HOP** | • 다음으로 거쳐야 할 **인접 라우터의 IP 주소**를 의미합니다.<br>• 업데이트 메시지를 보낼 때 다음 홉 주소를 명시하여 경로를 안내합니다. |
| **LOCAL_PREF** | • **지역 선호도**를 의미하며, 내부 네트워크에서 외부로 나가는 경로 중 어디를 선호할지 결정하는 매트릭 값입니다.<br>• AS-PATH나 NEXT_HOP보다 **경로 결정에 더 큰 영향**을 미치며, 관리자의 '정책'이 가장 많이 반영되는 속성입니다. |


## ⛓️ 전송 계층(Transport Layer)의 개요

전송 계층은 네트워크 계층(IP)과 응용 계층 사이에 위치하며, 두 가지 핵심 기능을 제공합니다.
1. **IP 프로토콜의 한계 극복:** 신뢰할 수 있는 통신과 연결형 통신을 지원합니다.
2. **애플리케이션 프로세스 식별:** 호스트 내에서 실행 중인 특정 프로그램을 식별합니다.

---

### ⚠️ 1. IP 프로토콜의 한계와 극복
IP 프로토콜은 전송 성능을 최우선으로 하기 때문에 본질적으로 아래와 같은 한계를 가집니다.

* **비신뢰성 (Unreliability):** 패킷이 목적지까지 제대로 전송되었는지 보장하지 않으며, 패킷 누락 시 재전송이나 순서 보장을 하지 않습니다.
* **비연결성 (Connectionless):** 송수신 호스트 간에 사전 연결 수립 과정을 거치지 않고 패킷을 일방적으로 전송합니다.

> **💡 해결책:** 전송 계층에서 신뢰성 있는 연결형 통신 프로토콜(**TCP**)을 제공하여 IP의 한계를 보완합니다.

---

### 🔄 2. 전송 계층의 두 가지 핵심 프로토콜

| 프로토콜 | 연결 방식 | 신뢰성 | 특징 |
| :--- | :--- | :--- | :--- |
| **TCP** | 연결형 통신 | **보장** (신뢰성 있음) | 데이터 분실 시 재전송, 전송 순서 보장 |
| **UDP** | 비연결형 통신 | **미보장** (비신뢰성) | 패킷 헤더가 가벼워 TCP보다 **비교적 빠른 송수신** 가능 |

## 🔌 포트(Port)의 개념과 유형

### 1. 포트란?
패킷이 특정 컴퓨터(호스트)에 도달하는 것을 넘어, 실행 중인 수많은 프로그램 중 **특정 프로그램(프로세스)으로 찾아가도록 식별하는 정보**입니다.
* **표기 형식:** 주로 `IP 주소 : 포트 번호` 형태로 묶어서 호스트와 프로그램을 동시에 식별합니다.
* **할당 범위:** 16비트로 표현되며, **0번부터 65,535번**까지 사용 가능합니다.

### 2. 포트 번호의 유형

| 포트 범위 | 유형 명칭 | 주요 특징 및 대표 예시 |
| :--- | :--- | :--- |
| **0 ~ 1,023** | **Well-Known Port**<br>(잘 알려진 포트 / 시스템 포트) | 전 세계적으로 범용 사용되는 프로토콜에 할당<br>• HTTP(80), HTTPS(443), DNS(53), FTP(21) 등 |
| **1,024 ~ 49,151** | **Registered Port**<br>(등록된 포트) | 대기업 제품이나 유명 오픈소스 프로토콜에 할당 |
| **49,152 ~ 65,535** | **Dynamic Port**<br>(동적 포트 / 사설 포트) | 특별한 관리 없이 클라이언트가 임시로 자유롭게 사용하는 포트 |

---

## 🌐 포트를 활용한 네트워크 기술 (NAT & 포트 포워딩)

### 1. NAPT (Network Address Port Translation)
기존 NAT는 공인 IP와 사설 IP를 1:1로 변환하여 IP 낭비 문제가 있었습니다. 이를 극복하기 위해 포트를 활용합니다.
* **동작 방식:** NAT 변환 테이블에 IP 주소뿐만 아니라 **포트 번호도 함께 기록하고 변환**합니다.
* **효과:** **하나의 공인 IP 주소를 여러 사설 IP 주소가 공유**할 수 있게 되어 공인 IP 주소 부족 문제를 크게 개선합니다.

### 2. 포트 포워딩 (Port Forwarding)
외부 호스트가 사설망 내부의 특정 서버나 호스트에 직접 접속하고자 할 때 사용하는 기술입니다.
포트 포워딩의 정의와 필요성
포트 포워딩은 네트워크 내부의 특정 호스트에 IP 주소와 포트 번호 쌍을 미리 할당하고, 해당 조합으로 들어오는 패킷을 지정된 호스트로 그대로 전달해 주는 기능이다.
외부 호스트는 공인 IP 주소와 포트 번호 조합만 알면 내부의 특정 호스트로 패킷을 전송할 수 있게 된다.

[외부 호스트] ───> [공인 IP : 포트 8080] ───> [공유기/라우터] ───> [내부 사설 IP : 포트 80 (지정된 호스트)]

* **필요성:** 내부 호스트들은 하나의 공인 IP를 공유하므로, 외부에서는 공인 IP만으로 내부의 특정 컴퓨터를 식별할 수 없습니다.
* **정의:** 외부에서 특정 포트 번호로 들어오는 패킷을 **내부의 특정 사설 IP와 포트 번호 쌍으로 그대로 전달(전송)**해 주도록 미리 이정표를 세워두는 기능입니다.


## 📡 ICMP (Internet Control Message Protocol)

### 1. ICMP의 정의와 역할
* **IP의 한계 보완:** IP 프로토콜의 본질적인 특성인 **비신뢰성**과 **비연결성**을 보완하기 위해 사용되는 프로토콜입니다.
* **피드백 제공:** 네트워크 장비(호스트, 라우터) 간에 주고받는 네트워크의 상태나 에러에 대한 **피드백 메시지**를 담당합니다.
* **구조:** ICMP 메시지는 네트워크의 상태 유형을 나타내는 **타입(Type)**과, 해당 유형의 세부 이유를 나타내는 **코드(Code)**의 조합으로 구성됩니다.

---

### 🔍 2. 네트워크 진단 정보 및 보조적 역할
ICMP는 단순한 오류 보고뿐만 아니라, 네트워크의 연결 상태를 진단하고 점검하는 보조적인 역할도 수행합니다.

* **에코 요청(Echo Request) & 에코 응답(Echo Reply):** 네트워크 진단의 대표적인 타입으로, 상대방 장비가 살아있는지 전송 경로에 문제가 없는지 확인합니다.
* **실무 활용 (`Ping` 명령어):** 우리가 흔히 터미널에 입력하는 `ping` 명령어가 바로 이 ICMP의 에코 요청과 응답 메커니즘을 기반으로 동작합니다.

| ICMP 타입 (Type) | 메시지 종류 | 주요 역할 및 용도 |
| :--- | :--- | :--- |
| **Type 8** | Echo Request (에코 요청) | 목적지 호스트에게 "연결이 가능한지" 확인하기 위해 패킷을 전송함 |
| **Type 0** | Echo Reply (에코 응답) | 에코 요청을 받은 목적지 호스트가 정상임을 알리기 위해 답변 패킷을 되돌려 보냄 |


## 🧩 TCP 통신의 과정 및 주요 기능

TCP 통신은 크게 세 가지 과정을 거칩니다.
1. **호스트 간의 연결 수립:** TCP는 연결 지향형 프로토콜이므로, 데이터 송수신 전에 반드시 연결이 수립되어야 합니다.
2. **데이터 송수신:** 수립된 연결을 바탕으로 데이터 송수신이 이루어지며, 이때 재전송을 통한 오류 제어, 흐름 제어, 혼잡 제어 등 높은 신뢰성을 보장하는 다양한 기능을 제공합니다.
3. **연결 종료:** 데이터 송수신이 끝나면 연결을 종료합니다.

### 📦 TCP 통신 단위 : MSS (Maximum Segment Size)
* MSS는 TCP 프로토콜로 전송할 수 있는 최대 세그먼트 크기를 나타내며, 특히 TCP 헤더 크기를 제외한 **최대 페이로드(데이터) 크기**를 의미합니다.

---

## 🩻 TCP 헤더 구조 및 핵심 필드

TCP 세그먼트 헤더는 송신지 포트, 수신지 포트, 순서 번호, 확인 응답 번호, 데이터 오프셋, 예약, 제어 비트, 윈도우, 체크섬 등 다양한 필드로 구성되어 있습니다. 이 중 송신지/수신지 포트, 순서 번호, 확인 응답 번호, 제어 비트(ACK, SYN, FIN 비트), 윈도우 필드는 매우 중요한 역할을 합니다.

* **포트 번호:** 현재 실행 중인 애플리케이션 프로세스를 식별하며, 16비트로 표현되어 0번부터 $2^{16}-1$ (65,535)까지의 값을 가집니다.

### 1. 순서 번호와 확인 응답 번호
* **순서 번호 (Sequence Number):** TCP를 통해 송수신되는 세그먼트 데이터의 첫 번째 바이트에 부여되는 번호로, 세그먼트의 올바른 순서를 보장하기 위해 사용됩니다. 초기 순서 번호는 무작위로 할당되며, 이후 세그먼트의 순서 번호는 초기 순서 번호에 송신된 바이트 수를 더한 값으로 결정됩니다.
* **확인 응답 번호 (Acknowledgment Number):** 수신자가 다음으로 수신하기를 기대하는 순서 번호를 나타냅니다. 일반적으로 수신한 순서 번호에 1을 더한 값이 확인 응답 번호가 됩니다. 확인 응답 번호를 포함한 세그먼트를 보낼 때는 제어 비트의 **ACK 플래그를 1로 설정**해야 합니다.

### 2. 제어 비트 (Control Bits)
제어 비트는 여러 비트의 조합으로 이루어져 있으며, 각 비트는 현재 세그먼트에 대한 부가 정보를 나타냅니다. 가장 중요한 핵심 비트는 다음과 같습니다.

* **`ACK` 비트:** 확인 응답 번호를 포함하는 세그먼트임을 나타냅니다.
* **`SYN` 비트:** 연결 수립 요청을 나타냅니다.
* **`FIN` 비트:** 연결 종료 요청을 나타냅니다.

## 🤝 TCP 연결 수립: 3-Way Handshake

TCP 프로토콜은 데이터 전송을 시작하기 전, 송수신 장치 간의 세션을 안전하게 연결하기 위해 **3-Way Handshake**라는 3단계 과정을 거칩니다.

### 🔄 3-Way Handshake 동작 과정

| 단계 | 플래그 (Flag) | 주요 동작 및 설명 |
| :--- | :---: | :--- |
| **1단계** | **`SYN`** | • 연결을 시작하는 호스트(**액티브 오픈**)가 상대방에게 연결 시작을 요청합니다.<br>• 세그먼트에 호스트 A의 **초기 순서 번호(ISN)**를 포함하여 전송합니다. |
| **2단계** | **`SYN + ACK`** | • 요청을 받은 호스트(**패시브 오픈**)가 연결 요청을 확인하고 응답합니다.<br>• 호스트 B의 초기 순서 번호와 함께, 호스트 A가 보낸 SYN에 대한 **확인 응답 번호(A의 초기 순서 번호 + 1)**를 담아 보냅니다. |
| **3단계** | **`ACK`** | • 액티브 오픈 호스트가 최종적으로 연결을 승인하는 단계입니다.<br>• 호스트 B가 보낸 SYN-ACK에 대한 확인 응답 번호를 담아 패킷을 보내며, 이 과정이 완료되면 **연결(Connection)이 수립**됩니다. |

> **💡 용어 직관하게 보기**
> * **액티브 오픈 (Active Open):** 먼저 능동적으로 연결을 시작하는 과정 (주로 클라이언트)
> * **패시브 오픈 (Passive Open):** 요청을 기다렸다가 수락하는 과정 (주로 서버)

## 🤝 TCP 연결 종료: 4-Way Handshake

데이터 송수신이 끝난 후, 수립되었던 세션을 안전하게 닫기 위해 **4-Way Handshake**라는 4단계 과정을 거칩니다.

1. **`FIN` (연결 종료 요청):** 연결 종료를 먼저 시작하는 호스트(**액티브 클로즈**)가 상대방에게 `FIN` 세그먼트를 보내 연결 종료를 요청합니다.
2. **`ACK` (수신 확인 및 자원 정리):** 요청을 받은 호스트(**패시브 클로즈**)는 `ACK` 세그먼트를 보내 `FIN` 세그먼트를 잘 받았음을 확인합니다. 이후 자신의 애플리케이션 종료 처리 및 남은 자원을 정리하는 대기 시간을 가집니다.
3. **`FIN` (종료 준비 완료 알림):** 자원 정리가 모두 끝난 패시브 클로즈 호스트는 상대방에게 `FIN` 세그먼트를 보내 자신도 연결을 종료할 준비가 되었음을 알린다.
4. **`ACK` (최종 종료 승인):** 액티브 클로즈 호스트는 패시브 클로즈 호스트의 요청에 응답하는 마지막 `ACK` 세그먼트를 보내며, 이 패킷이 전달되면 최종적으로 연결이 완전히 종료됩니다.

> **💡 용어 직관하게 보기**
> * **액티브 클로즈 (Active Close):** 연결 종료를 능동적으로 먼저 요청하는 측 (주로 클라이언트)
> * **패시브 클로즈 (Passive Close):** 종료 요청을 받고 자원을 정리한 뒤 수동적으로 종료하는 측 (주로 서버)

## ⚖️ 로드밸런서와 방화벽: 4계층 장비(세션 장비)

### 1. OSI 4계층 장비의 특징
* **핵심 제어 대상:** 4계층은 **TCP, UDP, Port(포트)**를 다루는 계층입니다.
* **판단 기준:** 4계층 장비는 **`IP + PORT`**의 조합을 보고 판단합니다.
  * *예시:* `192.168.0.10:80` (HTTP 웹 트래픽), `192.168.0.10:22` (SSH 원격 접속)
* **특징:** 어떤 서비스인지, 어떤 세션인지, 현재 누구와 누구가 연결되었는지 명확히 판단할 수 있습니다.

> **💬 핵심 용어 정리**
> * **세션 (Session):** 두 장비 간의 통신이 시작되어 종료될 때까지의 **연결 상태**를 의미합니다.
> * **세션 테이블 (Session Table):** 현재 누가 누구랑 연결되어 통신 중인지를 기록하고 저장하는 표입니다.

---

### 2. 주요 4계층 네트워크 장비

| 장비/기술명 | 주요 역할 및 핵심 기능 |
| :--- | :--- |
| **로드밸런서<br>(Load Balancer)** | • 특정 서버에 트래픽이 몰리지 않도록 **여러 대의 서버로 부하를 나누어주는 장비**입니다. |
| **L4 스위치** | • 4계층 정보인 **IP와 Port 번호를 기반**으로 트래픽을 효율적으로 분산하는 장비입니다. |
| **Health Check** | • 로드밸런서가 뒤에 있는 **서버들이 정상적으로 살아있는지 주기적으로 검사**하는 기능입니다. (죽은 서버에는 트래픽을 보내지 않음) |
| **ADC**<br>(Advanced Delivery Controller) | • L4 기능을 넘어 상위 계층인 **HTTP 내용(URL, 쿠키, 헤더 등)까지 상세하게 분석**하여 트래픽을 제어하는 고성능 장비입니다. |
| **방화벽<br>(Firewall)** | • 미리 정의된 **허용/차단 정책(Rules)**을 바탕으로 비인가 트래픽을 막고 내부 네트워크를 보호하는 보안 장비입니다. |


## 🌐 통신을 도와주는 네트워크 주요 기술

---

### 🔄 1. NAT / PAT (IP 주소 변환 기술)

* **NAT란?** Network Address Translation 즉, **IP 주소 변환 기술**입니다. 내부망에서는 사설 IP를 사용하다가 인터넷(외부망)으로 나갈 때만 공인 IP로 변환합니다.

#### 📌 NAT 동작 방식
1. **1단계:** `192.168.0.10`에서 웹 접속 시도
2. **2단계:** 출발지(`192.168.0.10`)에서 목적지(`142.x.x.x` - Google)로 패킷 생성
3. **3단계:** 공유기가 출발지 IP를 공인 IP로 변경 수행
4. **4단계:** 인터넷 전송
5. **5단계:** Google 응답
6. **6단계:** NAT 테이블에 변경 내역 저장

#### 📌 PAT 동작 방식
* 회사 내 PC는 100대인데 사용할 수 있는 공인 IP는 단 1개뿐일 때, 이를 해결하기 위해 **IP 주소와 포트(Port) 번호까지 함께 변환**하는 **PAT**가 등장했습니다.

#### 📌 SNAT와 DNAT
* **SNAT (Source NAT):** 출발지 주소를 변경하는 방식이며, 가장 흔하게 사용되는 일반적인 NAT입니다.
* **DNAT (Destination NAT):** 목적지 주소를 변경하는 방식이며, 외부에서 내부 서버로 접속을 허용하는 **포트 포워딩**에 사용됩니다.

#### 📌 동적 NAT와 정적 NAT
* **정적 NAT:** 공인 IP와 사설 IP를 **고정으로 1:1 변환**합니다.
* **동적 NAT:** 공인 IP 풀(Pool)을 관리하며, 내부 요청 시 **남는 공인 IP를 자동으로 할당**합니다.

### 🗺️ 2. DNS (Domain Name System)

* **DNS란?** 문자로 된 **도메인 이름을 숫자로 된 IP 주소로 변환**해 주는 시스템입니다. DNS가 없으면 매번 `223.130.195.200` 같은 복잡한 IP 주소를 직접 외워서 접속해야 합니다.

#### 📌 DNS 구조와 명명 규칙
DNS는 뿌리(`.`)부터 시작하는 명확한 **계층 구조(트리 구조)**를 가집니다.
* *예시:* `www.google.com`
  * **`com`:** 최상위 도메인 (Top-Level Domain)
  * **`google`:** 도메인 이름 (Domain)
  * **`www`:** 호스트 네임 (Host)
* **트리 구조 조회 방향:** `.` (루트) ➡️ `com` ➡️ `google` ➡️ `www`

#### 📌 DNS 동작 방식
사용자가 브라우저에 `google.com`을 입력하면 PC가 DNS 서버에 순차적으로 질문을 던져 IP를 찾아냅니다.

```text
 PC (사용자)
   │
   ▼
 로컬 DNS 서버
   │
   ├─► Root DNS 서버 (.)
   ├─► .com DNS 서버
   └─► google DNS 서버 ──► "IP 주소 변환 완료!"
   │
   ▼
[획득한 IP를 바탕으로 브라우저가 TCP 연결 시작]
```

#### 📌 DNS 캐싱 (매우 중요)
* 한 번 조회한 결과는 네트워크 성능 향상과 서버 부하 감소를 위해 시스템에 잠시 저장해 둡니다.
* **TTL (Time To Live):** 캐시가 메모리에 유지되는 유효 기간(유지 시간)을 의미합니다.

#### 📌 마스터와 슬레이브 (Master & Slave)
* 만약 운영 중인 DNS 서버가 다운되면 사용자는 사이트에 접속할 수 없게 됩니다. 
* 이를 방지하기 위해 원본 데이터를 가진 **Master 서버**와 복제본을 실시간으로 동기화하는 **Slave 서버**를 함께 구성합니다.
* **장점:** 서버 장애 발생 시 즉각적인 대응(이중화)이 가능하며, 요청을 나누어 처리하는 부하 분산 효과를 얻을 수 있습니다.

#### 📌 DNS 주요 레코드 종류
주로 사용되는 핵심 DNS 레코드의 유형과 역할은 다음과 같습니다.

| 레코드 유형 | 역할 및 변환 대상 |
| :---: | :--- |
| **A** | 도메인 이름을 **IPv4 주소**로 매핑합니다. |
| **AAAA** | 도메인 이름을 **IPv6 주소**로 매핑합니다. |
| **CNAME** | 도메인 이름에 대한 고유한 **별칭(Alias)**을 지정합니다. |
| **MX** | 해당 도메인과 연결된 **메일 서버**의 정보를 지정합니다. |
| **NS** | 도메인의 구체적인 정보를 관리하는 권한이 있는 **네임 서버(DNS 서버)**를 지정합니다. |

#### 📌 hosts 파일
* **역할:** 사용자 컴퓨터 로컬에 존재하며, **DNS 조회보다 무조건 최우선으로 적용되는 강제 매핑 파일**입니다.
* *예시:* `192.168.0.10 myserver.com`
* **사용 이유:** 로컬 테스트, 개발 환경 검증, 임시 서버 연결 등의 목적으로 사용됩니다.

---

### 🌍 3. GSLB (Global Server Load Balancing)

* **개념:** 쉽게 말해 지리적으로 멀리 떨어진 **전 세계 단위의 로드밸런싱 기술**입니다.
* **흐름:** 사용자가 도메인을 요청하면 GSLB가 위치를 판단하여 **네트워크상 가장 가깝거나 상태가 가장 좋은 서버로 연결**해 줍니다.
* **클라우드 실무:** 클라우드 환경에서 고가용성과 장애 복구(DR)를 위해 매우 중요하게 다뤄집니다. (예: AWS Route 53, Cloudflare 등)

---

### 🔌 4. DHCP (Dynamic Host Configuration Protocol)

* **역할:** 네트워크망 내의 장비들에게 **IP 주소를 자동으로 할당**해 주는 프로토콜입니다.
* **장점:** DHCP가 없으면 관리자가 모든 기기에 IP를 수동으로 입력해야 하지만, DHCP가 있으면 **PC를 켜자마자 자동으로 완벽한 네트워킹 설정**이 완료됩니다.
* **DHCP가 할당해 주는 핵심 정보 4가지:**
  1. **IP 주소:** 호스트 고유 주소
  2. **서브넷 마스크:** 네트워크의 범위
  3. **기본 게이트웨이:** 외부 인터넷으로 나가는 출구
  4. **DNS 서버 IP:** 도메인 이름 해석을 요청할 서버 주소

#### 📌 DHCP 동작 방식 (DORA 과정)
DHCP는 IP 주소를 자동으로 임대하기 위해 총 4단계의 유기적인 브로드캐스트/유니캐스트 과정을 거쳐 동작합니다.

```text
 PC (클라이언트)                                    DHCP 서버
     │                                                 │
     ├─────── [1] Discover (발견) ────────────────────►│ (네트워크 전체에 "거기 DHCP 서버 있나요?")
     │                                                 │
     ◄─────── [2] Offer (제안) ────────────────────────┤ ("이 IP 주소 대역을 사용해 보세요.")
     │                                                 │
     ├─────── [3] Request (요청) ─────────────────────►│ ("좋습니다! 제안하신 그 IP 제가 쓰겠습니다.")
     │                                                 │
     ◄─────── [4] ACK (승인) ──────────────────────────┤ ("확인되었습니다. 최종 할당을 승인합니다.")
     │                                                 │

```

## 🖥️ 서버의 네트워크 설정 및 확인

* **개념:** 서버는 WEB, WAS, DB 등의 역할을 수행하지만, 기본적으로 고유한 IP를 가지고 통신하는 **네트워크 장비**입니다.

### 1. OS별 네트워크 정보 확인 명령어

| 확인 대상 | 🐧 리눅스 (Linux) | 🪟 윈도우 (Windows) |
| :--- | :--- | :--- |
| **IP 주소 확인** | `ip addr` 또는 `ifconfig` | `ipconfig` 또는 `ipconfig /all` |
| **게이트웨이 확인** | `ip route` | `ipconfig` |
| **DNS 설정 확인** | `cat /etc/resolv.conf` | `ipconfig /all` |

---

### 2. 서버의 라우팅 테이블 (Routing Table)

* **라우팅 테이블이란?** 서버가 패킷을 어디로 보내야 하는지 기록해 둔 **네트워크 지도**입니다.
  * **같은 네트워크 대역:** 게이트웨이를 거치지 않고 목적지로 **직접 전송**합니다.
  * **다른 네트워크 대역:** 외부망으로 나가기 위해 지정된 **게이트웨이(출구)로 전송**합니다.
* **디폴트 라우트 (Default Route):** 목적지 경로를 알 수 없는 모든 외부 트래픽을 보낼 기본 방향 정보입니다. (`0.0.0.0/0`)

#### 📌 OS별 라우팅 테이블 확인 명령어
* **리눅스:** `ip route` 또는 `route -n`
* **윈도우:** `route print`

---

### 3. 필수 네트워크 확인 명령어 요약

| 명령어 | 주요 역할 및 활용 방법 |
| :--- | :--- |
| **`ping`** | • **가장 기본 장비 활성화 확인:** 상대 장비가 살아있는지 검사합니다. (ICMP 프로토콜 사용)<br>• *예시:* `ping 8.8.8.8` (구글 DNS 서버와 네트워크 연결 상태 점검) |
| **`tcping`** | • **특정 TCP 포트 오픈 확인:** 윈도우 환경에서 특정 서비스 포트가 열려있는지 확인합니다.<br>• *예시:* `tcping google.com 443` (HTTPS 포트 연결 가능 여부 체크) |
| **`traceroute`<br>/ `tracert`**| • **목적지까지의 경로 추적:** 패킷이 거쳐가는 라우터들을 추적하여 어느 구간에서 장애가 발생하는지 위치를 찾아냅니다.<br>• *예시:* `traceroute google.com` (Linux), `tracert google.com` (Windows) |
| **`netstat`** | • **현재 네트워크 연결 상태 확인:** 활성화된 TCP 세션 상태, 서버가 대기 중인 `LISTEN` 포트, 어떤 프로세스가 포트를 쓰고 있는지 확인합니다.<br>• *예시:* `netstat -ant` (`a`: 전체, `n`: 숫자로 표시, `t`: TCP 필터링) |
| **`ss`** | • **현대 리눅스의 netstat 대체 명령어:** `netstat`보다 처리 속도가 빨라 최신 리눅스에서 자주 권장됩니다.<br>• *예시:* `ss -ant` |
| **`nslookup`** | • **DNS 질의 및 조회:** 특정 도메인에 연결된 IP 주소가 무엇인지 DNS 서버에 물어보고 확인할 때 사용합니다.<br>• *예시:* `nslookup google.com` |
| **`telnet`** | • **포트 오픈 수동 테스트:** 원래는 원격 접속 도구이지만, 방화벽이 열려있는지 포트 테스트용으로 널리 쓰입니다.<br>• *예시:* `telnet 192.168.0.10 80` (80번 웹 포트 접근 테스트) |
| **`tcpdump`** | • **리눅스 패킷 분석의 핵심:** 네트워크 인터페이스로 오고 가는 실제 패킷 흐름을 실시간으로 캡처하고 감시합니다.<br>• *예시:* `tcpdump -i eth0` (`eth0` 카드의 패킷 모니터링) |


## 🛡️ 네트워크 정보 보안

---

### 1. 보안의 개념과 정의
* **정보 보안의 정의:** 정보를 여러 가지 잠재적 위협으로부터 안전하게 보호하는 것을 의미합니다.

#### 📌 보안의 핵심 3요소 (CIA)
네트워크와 시스템 보안을 구축할 때 반드시 충족해야 하는 세 가지 필수 요소입니다.

| 보안 요소 | 영문명 | 핵심 개념 및 정의 | 대표 기술 / 주요 위협 |
| :---: | :--- | :--- | :--- |
| **기밀성** | **C**onfidentiality | • 오직 **허가된 사용자만** 데이터에 접근할 수 있도록 통제하는 원칙입니다. | • 암호화, 접근제어(ACL), VPN |
| **무결성** | **I**ntegrity | • 전송되거나 저장된 데이터가 정당한 방법 외에 **임의로 위변조되지 않음**을 보장합니다. | • 해시(Hash) 함수, 디지털 서명 |
| **가용성** | **A**vailability | • 권한이 있는 사용자가 **필요할 때 언제든지 서비스를 상시 이용**할 수 있어야 합니다. | • **주요 위협:** DDoS 공격 (서버 다운 및 장애 유발) |

### 2. 네트워크 정보 보안 개요

* **네트워크 보안이란?** 네트워크망을 통해 들어오는 다양한 형태의 사이버 공격을 선제적으로 방어하고 내부 자산을 보호하는 기술입니다.
* **대표적인 공격 유형:** * **해킹:** 서버에 비인가 접속하여 제어권을 탈취하는 행위
  * **스니핑:** 네트워크 트래픽을 가로채어 데이터를 몰래 훔쳐보는 도청 행위
  * **DDoS:** 대량의 트래픽을 유발하여 서버나 네트워크 자원을 마비시키는 공격
  * **악성코드:** 시스템에 피해를 주기 위해 제작된 바이러스, 웜 등의 소프트웨어
  * **웹 공격:** SQL Injection 등 웹 애플리케이션의 취약점을 노린 공격
* **보안의 기본 원칙:** 정상적인 외부 사용자의 접근은 안전하게 허용하되, 악의적인 공격자의 트래픽은 철저히 차단하기 위해 **방화벽, IPS, WAF, VPN** 등의 전문 보안 장비를 계층적으로 배치합니다.

### 3. 보안 솔루션 종류와 흐름

#### 📌 보안 솔루션 인라인(Inline) 전체 구조
외부 인터넷 망에서 들어온 트래픽은 서비스 서버(WEB)에 도달하기 전까지 아래와 같은 순서로 정밀 검사 단계를 거칩니다.

```text
인터넷 ──> DDoS 방어 장비 ──> 방화벽 ──> IPS ──> WAF ──> WEB 서버
```

#### 📌 주요 보안 솔루션별 상세 역할

| 솔루션 명칭 | 주요 역할 및 핵심 기능 |
| :--- | :--- |
| **DDoS 방어 장비** | • **DDoS (Distributed Denial of Service):** 엄청난 양의 분산 트래픽을 유발하여 서버를 마비시키는 공격입니다.<br>• **역할:** 네트워크 인입 단계에서 비정상적인 트래픽을 정밀하게 필터링하여 제거합니다. |
| **방화벽 (Firewall)** | • **통신 제어:** 사전에 정의된 보안 정책(Rule)에 따라 허용할 통신과 차단할 통신을 명확히 결정합니다.<br>• *예시 정책:* 허용(80-HTTP, 443-HTTPS) / 차단(23-Telnet, 기타 불필요한 포트 전체) |
| **IDS / IPS** | • **침입 탐지 및 방지:** 패킷 내부 데이터를 분석하여 수상한 공격 징후를 실시간 감시합니다.<br>• `IDS (Intrusion Detection System)`: 공격 발견 시 **탐지 및 알림**만 수행 (차단 X)<br>• `IPS (Intrusion Prevention System)`: 공격 발견 시 **탐지와 차단**을 즉시 동시에 수행 |
| **WAF**<br>(Web Application Firewall) | • **웹 애플리케이션 방화벽:** 일반 방화벽과 달리, HTTP/HTTPS 웹 프로토콜의 7계층 콘텐츠 내용을 직접 열어보고 웹 공격(SQL Injection 등) 자체를 정밀 분석하여 방어합니다. |
| **샌드박스 (Sandbox)** | • **가상 격리 환경 분석:** 악성 여부가 의심되는 파일이나 메일 첨부파일을 외부와 격리된 가상 환경에서 먼저 실행해 본 뒤, 악성 행위 여부를 판단하여 내부 반입을 통제합니다. |
| **NAC**<br>(Network Access Control) | • **네트워크 접근 제어:** 사전에 인증을 통과했거나 무결성 검증(백신 설치 등)을 마친 **허가된 장비만** 내부 사내 네트워크에 접속할 수 있도록 통제합니다. |

#### 📌 네트워크 통제 메커니즘
* **IP 제어:** 사전에 정의된 특정 IP 또는 IP 대역에 대해서만 접근을 명시적으로 허용하거나 차단합니다.
* **접근 통제:** 인증된 사용자가 직무와 권한에 따라 어떤 서버의 어떤 데이터까지 접근할 수 있는지 세부적인 통로와 권한을 제어합니다.
* **VPN (Virtual Private Network):** 외부에서 회사 내부망에 안전하게 원격 접속할 수 있도록 구성하는 가상 사설망입니다. 
  * *핵심 메커니즘:* 데이터를 안전하게 보호하는 **암호화** 기술과 외부 노출을 방지하는 **터널링(안전한 통신 경로)** 기술이 결합되어 동작합니다.

#### 📌 방화벽 (Firewall) 정의 및 발전 과정

> **방화벽 정의:** 미리 정의된 보안 정책(Rule)을 기반으로 네트워크 트래픽의 **허용(Allow) 및 차단(Drop)** 정책을 수행하는 가장 기본적인 네트워크 보안 장비입니다.

| 분류 | 특징 및 핵심 메커니즘 | 단점 및 한계 |
| :---: | :--- | :--- |
| **초기 방화벽**<br>(Packet Filtering) | • 단순 패킷 헤더 정보(출발지/목적지 IP, Port)만 확인하여 차단 여부를 결정합니다. | • **연결 상태(State)를 인지하지 못함:** 패킷이 정당한 통신 흐름 안에서 오고 가는 것인지 구별할 수 없습니다. |
| **현대적 방화벽**<br>(SPI) | • **Stateful Packet Inspection (세션 상태 기억):** 통신 세션의 상태를 테이블에 기록하고 지속적으로 추적합니다.<br>• *예시:* `TCP 3-Way Handshake` 과정을 정상적으로 거친 신뢰할 수 있는 연결인지 검사합니다. | • **장점:** 세션의 흐름을 꼬아서 들어오는 비정상적인 우회 세션 공격을 효과적으로 탐지하고 차단할 수 있습니다. |

#### 📌 방화벽 동작 방식 프로세스
네트워크 패킷이 방화벽에 인입되면 다음과 같은 흐름을 거쳐 처리됩니다.

```text
[ 패킷 수신 ]
     │
     ▼
[ 세션 테이블 확인 ] ───► (기존에 연결된 정상 세션인가?) ───► [ 즉시 허용 ]
     │ (기존 세션 없음)
     ▼
[ 방화벽 정책 검사 ] ───► (새로운 연결이 허용된 정책인가?)
     │
     ├─► 허용 (Allow) ──► 세션 테이블에 기록 후 패킷 통과
     └─► 차단 (Drop)  ──► 패킷 폐기 및 통신 차단

```

#### 📌 ALG (Application Layer Gateway)
* **정의:** 애플리케이션 계층(7계층) 프로토콜 데이터까지 깊게 분석하여 NAT 및 방화벽 통과를 보조하는 기술입니다.
* *예시:* 데이터 채널을 제어 채널과 별개로 생성하는 특성을 가진 **FTP 프로토콜**처럼, 일반적인 NAT 환경에서 통신이 끊어지기 쉬운 특수 프로토콜의 정상적인 통신을 도와줍니다.

#### 📌 IPS / IDS 및 차세대 기술
* **동작 방식:** 패킷 내부 데이터를 뜯어보는 패턴 분석(시그니처 매칭)과 정상 범위를 벗어나는 행위를 잡아내는 이상 탐지 기법을 수행합니다.
* **기존의 한계:** 기존 IPS는 이미 알려진 공격 패턴만 막을 수 있어 신종 공격(Zero-Day) 대응에 약하다는 단점이 있습니다.
* **NGIPS (Next-Generation IPS) 등장:** 단순 패턴 검사를 넘어 **애플리케이션 분석, 사용자 권한 식별, 정밀 행위 기반 탐지** 기능이 대폭 강화되어 고도화된 위협을 방어합니다.

#### 📌 DDoS 방어 장비 및 주요 위협
* **공격 목적:** 서버가 처리할 수 없는 막대한 자원이나 트래픽을 유발하여 시스템 가용성을 파괴하고, 서비스를 강제로 마비시키는 것입니다.
* **대표적인 DDoS 공격 유형:**
  * `SYN Flood`: 무수한 TCP 연결 요청(SYN) 패킷을 보내 서버의 연결 대기 자원을 고갈시키는 공격 (TCP 연결 폭탄)
  * `UDP Flood`: 대량의 UDP 트래픽을 한꺼번에 유발하여 네트워크 대역폭을 가득 채우는 공격 (UDP 트래픽 폭탄)
  * `HTTP Flood`: 정상적인 웹 페이지 요청을 과도하게 발생시켜 웹 서버의 처리 자원을 고갈시키는 공격 (웹 요청 폭탄)

#### 📌 VPN (Virtual Private Network) 메커니즘
* **개념:** 공용 인터넷망에 암호화 터널을 생성하여, 외부에서도 마치 사내 내부망에 직접 연결된 것처럼 안전하게 통신할 수 있도록 돕는 가상 사설망 기술입니다.
* **통신 흐름:**
```text
사용자 ──> [데이터 암호화] ──> 인터넷 (공용망) ──> 회사 VPN 서버 [복호화] ──> 사내 서버 도달
```
