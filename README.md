# 💻 컴퓨터 네트워크 공부 요약
공부한 네트워크 핵심 이론과 실습 내용을 기록하는 공간입니다.

------

## 🌐 네트워크 첫걸음

* **네트워크란?** 컴퓨터간의 연결이 되어있고 데이터를 주고받는 것을 컴퓨터 네트워크라고 한다.
* **인터넷이란?** 전세계 크고 작은 네트워크들이 연결된 거대한 네트워크.
* **패킷이란?** 컴퓨터간의 데이터를 주고받을 때 네트워크를 통해 흘러가는 작은 데이터 조각이다.
* **디지털 데이터:** 컴퓨터는 `0`과 `1`만 아는데, 이 `0`과 `1`의 집합을 디지털 데이터라고 한다.
  * **비트(bit):** 0과 1의 정보를 나타내는 최소 단위.
  * **바이트(Byte):** 8비트(bit) = 1바이트(Byte).
* **LAN이란?** Local Area Network, 건물 안이나 특정 지역을 범위로 하는 네트워크.
* **WAN이란?** Wide Area Network, ISP가 제공하는 서비스를 사용하여 구축한 네트워크.
* **DMZ란?** 외부에 공개하기 위한 네트워크.
* **온프레미스(On-Premise):** 사내 또는 데이터센터에 서버를 두고 직접 운영하는 것 (반대로는 클라우드가 있음).

## 🌐 크기에 따른 네트워크 분류 : LAN 과 WAN
* **LAN이란?** Local Area Network, 건물 안이나 특정 지역을 범위로 하는 네트워크.
* **WAN이란?** Wide Area Network, ISP가 제공하는 서비스를 사용하여 구축한 네트워크.

## 🌐 연결 형태에 따른 분류 : 네트워크 토폴로지
* 네트워크는 연결 형태에 따라 `스타형(성형)` 과 `메시형(망형)`으로 나눌 수 있다.
* 스타형 : 중앙 장비에 모든 장비가 연결된 형태, 장점 : 장애 찾기 쉽다, 관리가 쉽다. 단점 : 중앙 스위치 죽으면 전체 영향이간다. 
* 메시형 : 여러 노드들이 서로 전부 연결된 형태이며 일부 장비가 고장나도 다른 경로를 통해 통신이 가능 WAN에서 주로 사용되는 방식, 안전성 높지만 비용이 높다
* 버스형 : 하나의 선 공유 과거에 주요사용됨
* 링형 : 원형 연결, 일부 특수 환경에서 사용한다.
* 대부분의 네트워크는 LAN과 WAN이 혼합된 형태이다. **

## 🌐 네트워크 통신 방식 : 유니캐스트, 멀티캐스트, 브로드캐스트
* 유니캐스트 : 특정 대상과만 통신하는 방식 ex) 특정 사람과 통신, 1대1 통신, 카카오톡, SSH, 웹서핑 등
* 멀티캐스트 : 특정한 다수와 통신하는 방식 ex) 특정 그룹과 통신, IPTV
* 브로드캐스트 : 같은 네트워크 대역에 있는 모든 사용자와 통신하는 방식 ex) 모든 사람과 통신, ARP 요청
* 애니캐스트 : 가장 가까운 서버로 전달 ex) DNS 서버

## 🌐 프로토콜 : 네트워크 통신의 약속
* 프로토콜 : 컴퓨터가 네트워크 망에서 특정 컴퓨터와 통신할 때는 프로토콜이라는 것을 사용한다. 프로토콜은 일종의 약속 또는 양식이다.
* 프로토콜에는 이더넷, ICMP, IP, TCP, UDP 등 여러 종류가 있으며, 각각 사용하는 네트워크 주소가 다르다.
* IP : 주소지정
* TCP : 신뢰성
* UDP : 속도
* HTTP : 웹 통신
* HTTPS : 암호화 웹
* DNS : 이름 변환
* ARP : IP -> MAC 변환
* ICMP : 네트워크 상태 확인

## 🌐 패킷의 구조와 역할
* 헤더, 페이로드, 트레일러 구조
* 패킷은 네트워크를 통해 주고받는 메시지의 기본 단위
* 헤더는 대부분 포함되지만, 트레일러는 없을 수도 있다.
* 헤더는 패킷의 송수신지 정보를 담고 있다.
* 1계층 bit
* 2계층 프레임
* 3계층 패킷
* 4계층 세그먼트

## 🌐 TCP/IP 모델 과 OSI 7 레이어 모델
* TCIP/IP 모델 : 네트워크 인터페이스, 네트워크, 전송, 응용 4개 계층으로 이루어져있다.
* OSI 7레이어 모델은 ISO에서 지정한 모델로, 통신 전체를 체계적으로 다룬다.
* OSI 7계층 : 응용계층, 표현계층, 세션계층, 전송계층, 네트워크계층, 데이터링크계층, 물리계층 (응표세전네데물)
  * 물리계층 : 전기 신호 전달, 케이블, 허브
  * 데이터링크 : MAC 주소 기반 통신, 스위치
  * 네트워크 : IP 주소 기반, 라우터
  * 전송 : TCP/UDP 담당, PDU:세그먼트
  * 세션 : 연결 관리
  * 표현 : 암호화/압축, SSL/TLS
  * 응용 : 사용자 프로그램, 브라우저, 메신저 / HTTP, FTP, SMTP
* 이더넷, IP, ICMP, ARP, TCP, UDP, HTTP 등의 프로토콜이 있다.
* TCP/IP가 더 실무적으로 되어있다.

## 🌐 캡슐화/디캡슐화
* 패킷은 페이로드(데이터), 헤더(제어정보), 푸터(꼬리말) 로 구성되는데 대부분의 경우 헤더만 사용된다.
* 페이로드에 프로토콜을 헤더로 붙이는 과정을 캡슐화(Encapsulation)이라고 한다.
* 캡슐화는 상위 계층에서 하위 계층으로 내려가면서 프로토콜을 붙이는 과정이다.
* 패킷을 받는 쪽에서 디캡슐화(Decapsulation)과정을 통해 패킷을 확인한다.
* 하위 프로토콜부터 하나씩 벗겨내며 데이터를 확인하는 과정으로, 누가 누구에게 어떤 데이터를 보냈는지 순차적으로 파악한다
* 하위계층에서부터 상위 계층으로 올라가며 프로토콜 헤더를 제거하고 최종적으로 순수한 데이터만 얻게 된다.
* 인캡슐레이션 (응용데이터 -> TCP Header(포트 번호 포함)  -> IP Header(IP주소포함) -> Ethernet Header(MAC주소포함) -> 전송)
* 디캡슐레이션 (Ethernet제거 -> IP 제거 -> TCP 제거 -> 응용데이터 도착

## 🌐 네트워크 연결 부분
* 네트워크는 유선, 무선 연결방식으로 나뉨
* 유선 네트워크 대표는 이더넷(Ethernet)
  * `이더넷 Ethernet`
     * 현재 LAN의 표준 기술
     * MAC 주소를 사용
     * 스위치 기반
     * 프레임 단위 통신
     * CSMA/CD 기반(과거 허브 환경) : 먼저 통신 중인지 확인 하고 충돌 나면 기다렸다 재전송 요즘은 스위치 사용으로 충돌 거의 없음
* 반이중 /전이중
   * 반이중(Half Duplex) 통신 : 한 번에 한 방향만 통신한다. ex) 무전기
   * 전이중(Full Duplex) 통신 : 동시에 송수신 가능하다. ex) 전화

 ## 🌐 네트워크 회선
 * 회선이란? 데이터가 이동하는 통로 ex) 인터넷 회선, 광케이블, 전용선
 * 회선의 종류
    * 전용 회선 : 기업에서 많이 사용
    * 인터넷 회선 : 일반 인터넷, 여러 사용자가 공유
    * 광케이블 : 빛으로 데이터를 전달, 빠르고, 노이즈적고, 장거리 가능하다.
  
 ## 🌐 네트워크 구성 요소
 * 허브 : 1계층 장비, 들어온 데이터를 모든 포트로 전달한다.
 * 스위치 : 2계층 장비, MAC 주소 기반 전달한다. MAC Address Table이 있다
 * 라우터 : 3계층 장비, 다른 네트워크와 연결, 라우팅 테이블
 * 로드밸런서 : 트래픽 분산 장비 : 서버 여러 대로 분산, 부하 분산, 장애 대응, 성능 향상
 * 방화벽 : 보안장비
 * IDS/ IPS : IDS 침입탐지, 알림중심 / IPS : 침입차단, 자동 대능 가능
 * VPN : 가상 사설망, 인터넷을 통해 안전하게 내부망 접속
 * NAT : 사설IP <-> 공인 IP 변환
 * DHCP : IP 자동 할당

  ## 🌐 MAC 주소
  * 랜카드(NIC)에 부여된 물리 주소
  * 2계층 주소
  * OO:1A:2B:3C:4D:5E / 48비트 사용
  * 전 세계 유일하며, 제조사 정보 포함되어있고, 변경 가능은 하지만 기본은 고유값이다.
  * MAC 주소는 앞 24비트는 제조사 뒤 24비트는 장비 고윳값이다.

 ## 🌐 IP 주소
  * 네트워크에서 사용하는 논리 주소
  * IPv4 형태, 192.168.0.1 32비트
  * 다른 네트워크 찾기 위해 필요하다.
  * 라우터가 IP 기반 동작한다.
  * 공인IP, 사설IP
     * 공인IP : 인터넷에서 유일한 주소(통신사 제공)
     * 사설IP : 내부망용 주소 / 192.168.x.x , 10.x.x.x, 172.16.x.x
     * 사설 IP는 IPv4 부족때문에 사용한다.
  * NAT : 사설 IP <-> 공인 IP 변환
  * 서브넷 마스크 : 네트워크/호스트 구분용도
  * CIDR 표기 : 앞 24비트가 네트워크 부분이다.
  * 기본 게이트웨이 : 외부로 나갈 출구
  * DNS : 도메인 -> IP 변환 ex) goolgle.com -> 142.x.x.x
  * IP 통신 과정 : 내 pc -> 게이트웨이 -> 라우터 -> 인터넷 -> 목적지 서버
  * TTL : Time To Live , 패킷 생존 시간, 라우터 지날 때 감소하며 0 되면 폐기, 루프 방지 목적이다.

 ## 🌐 TCP와 UDP
  * TCP 는 신뢰성 중심, 연결형, 순서보장, 재전송, 흐름제어 , 혼잡제어
  * TCPsms 3-Way Handshaek 로 연결
     * SYN, SYN-ACK, ACK
  * TCP는 안정적이고 데이터 손실 복구가 되나 느리다. 주로 웹, 로그인, 파일전송 , DB에서 사용한다.
  * UDP는 속도 중심이고 비연결형, 빠르다, 신뢰성없다, 순서보장 하지 않는다.
  * 주로 게임, 스트리밍 에서 사용한다.

  ## 🌐 이더넷의 이해와 중요성
   * 이더넷은 현대 LAN 환경, 특히 유선 LAN에서 가장 널리 사용되는 기술이다.
   * 물리 계층에서의 통신 매체 간 정보 교환 규칙과 데이터 링크 계층에서 두 호스트 간 메시지 교환 형식을 정의하는 핵심 기술이다.
   * 오늘날 대부분의 유선 LAN 환경은 이더넷 기술을 기반으로 구성된다.
   * 물리 계층의 케이블과 데이터 링크 계층에서 주고받는 프레임이 모두 이더넷 규격을 따른다.

  ## 🌐 이더넷 프레임
   * 데이터 링크 계층에서 메시지를 주고받는 단위는 `프레임`이라고 한다.
   * 이더넷 네트워크에서 사용되는 프레임을 이더넷 프레임 이라 한다.
   * 이더넷 프레임은 헤더, 페이로드(데이터), 트레일러로 구성된다.
   * 이더넷 프레임의 헤더는 크게 프리앰블, 수신지 MAC 주소(6바이트), 송신지 MAC 주소(6바이트), 타입(2바이트), 길이 필드로 구성된다.
   * 프리앰블은 8바이트 길의 비트열로, 수신지에서 프레임 수신을 준비하도록 동기화하는 역할을 한다.
   * MAC 주소는 수신지 및 송신지 MAC 주소 필드에 명시되는 네트워크 인터페이스마다 부여되는 6바이트(48비트) 길이의 물리적 주소이다.

  ## 🌐 NIC(네트워크 인터페이스 컨트롤러)란?
   * NIC는 호스트와 통신 매체를 연결하는 인터페이스 역할을 수행하는 장비이며 흔히 네트워크 카드, 랜 카드 등으로 불린다.
   * NIC는 통신 매체의 전기신호를 호스트가 이해할 수 있는 정보로 변환하거나, 호스트의 디지털 정보를 통신 매체 신호로 변환하는 역할을 한다.
   * 호스트가 네트워크를 통해 송수신하는 모든 정보는 NIC를 거치게 된다.
   * NIC는 MAC 주소를 인식하여 수신한 프레임의 송수신지 MAC 주소를 확인하고, 자신과 관련 없는 프레임을 폐기하거나 일치하는 프레임을 역캡슐화 수행한다.

  ## 🌐 물리 계층과 데이터 링크 계층 장비의 핵심 차이
   * 물리 계층의 네트워크 장비는 주소 개념이 없어 송수신되는 정보에 대한 조작이나 판단을 할 수 없다.
   * 데이터 링크 계층 이상의 장비는 MAC 주소와 같은 주소 개념을 가지고 있어 송수신 정보를 특정하고 판단하여 조작할 수 있다.
  
  ## 🌐 물리계층 대표 장비 `허브` (1계층)
   * 허브는 `리피터 허브` 또는 `이더넷 허브` 등으로 불리며, 여러 호스트를 연결하는데 사용된다.
   * 허브의 첫 번째 전송 특징은 모든 포트로 해당 메시지를 내보낸다. 이러한 것은 불필요한 트래픽을 과도하게 증가시키는 문제점을 야기시킨다.
   * 허브의 두 번재 특징은 반이중 방식으로 통신한다. ex) 무전기
   * 모든 포트로 신호를 보내고 반이중 방식으로 통신하는 특성 때문에 충돌(Collision)문제가 발생한다. 충돌이 발생할 수 있는 영역을 콜리전 도메인 이라고 한다.
   * 충돌을 줄이려고 CSMA/CD 프로토콜을 이용한다.

  ## 🌐 데이터링크계층 대표 장비 `스위치` (2계층)
   * 스위치는 허브와 달리 어떤 MAC 주소를 가진 호스트가 어떤 포트에 연결되어 있는지 식별할 수 있다.
   * 스위치는 전 이중 모드 통신을 지원하여 근본적으로 CSMA/CD 프로토콜이 필요가 없다.
   * 일반적으로 스위치라하면 L2 스위치를 말하긴한다.
   * 스위치의 주요기능으로는 첫 번째는 `MAC 주소 학습 기능`이다.
      * 이 기능은 어떤 포트에 어떤 MAC 주소를 가진 호스트가 연결되어 있는지 학습하여 기억한다.
      * 스위치가 전달받은 신호를 원하는 목적지 MAC 주소가 연결된 포트로만 내보낼 수 있게 해준다.
      * MAC 주소 학습을 위해 스위치는 MAC 주소 테이블이라는 특별한 정보를 저장한다.
      * 이 테이블은 스위치 포트와 연결된 호스트의 MAC 주소 간의 연관 관계를 나타내는 표와 같으며, 스위치는 이 테이블을 통해 특정 포트에 연결된 호스트의 MAC 주소를 알 수 있다.
      * MAC 주소 테이블은 플러딩, 포워딩, 필터링, 에이징 을 통해 생성되고 관리된다.
      * `스위치 플러딩`은 처음에 MAC 주소 테이블이 비어있어 호스트의 위치를 모르면 허브처럼 모든 포트로 프레임을 전송한다.
      * MAC 주소를 학습한 이후에는 A->C로 전송할 때 다른 포트로 보내지 않는다 이것을 `필터링`이라고한다.
      * 연결된 호스트에 포트로만 프레임을 내보내는것을 `포워딩`이라고 한다.
      * `에이징`은 MAC 주소 테이블에 있는 항목을 일정 시간이 지나면 삭제하는 기능이다.
    
   * 스위치의 두 번째 주요 기능은 `VLAN(Virtual LAN)` 이다.
      * VLAN은 논리적으로 나누어진 가상의 LAN을 의미하며, 한 대의 스위치로 여러 개의 가상 LAN을 만들 수 있게 해준다.
      * VLAN을 이용하면 불필요한 트래픽을 분배하여 성능 저하를 방지할 수 있다.
      * VLAN을 사용하면 물리적인 스위치가 한 대라도 여러 대의 스위치가 있는 것처럼 호스트의 물리적 위치와 관계없이 논리적인 단위로 LAN을 구성하고 나눌 수 있다.

   ## 🌐 네트워크 계층 (3계층)
 * 네트워크 계층과 IP 프로토콜의 중요성
   * 네트워크 계층은 네트워크 기능의 핵심이다.
   * IP 프로토콜이 가장 중요한 역할을 수행한다.
   * IP는 물리 계층과 데이터 링크 계층의 한계를 극복하고, IP 주소 지정과 라우팅을 가능하게 한다.
   * IP 프로토콜에는 v4 와 v6가 있지만 기본적으로 ipv4를 의미한다.
 
 * IP 프로토콜의 공식 기능 : 주소 지정과 단편화
   * 주소 지정은 IP주소를 사용하여 특정 네트워크의 호스트를 식별하는 것이다.
   * 단편화는 큰 패킷을 여러 작은 패킷으로 나누어 전송하는 과정을 의미한다.
 
 * IP 주소 지정 상세 설명
   * IP 주소 지정은 IP 주소 체계를 통해 호스트가 속한 네트워크와 호스트 자체를 식별한다.
   * IPv4 주소는 32비트로 구성되며, 4개의 옥텟으로 표현된다.
   * 각 옥텟은 0부터 255까지의 숫자로 이루어져 있다.

 * IP 단편화 과정 및 MTU의 역할
   * 단편화는 전송하려는 패킷 크기가 MTU보다 클 때 발생한다.
   * MTU는 한 번에 전송 가능한 IP 패킷의 최대 크기를 의미하며, 일반적으로 1500바이트 이다.
   * 큰 데이터가 MTU이하의 패킷으로 쪼개져 전송되면, 수신 측에서는 이 패킷들을 재조합해야 한다.

 * IPv4 패킷 헤더 주요 필드
   * 중요한 필드로는 식별자, 플래그, 단편화 오프셋, TTL, 프로토콜, 송신지 IP 주소,수신지 IP주소가 있다.
   * 식별자 필드는 단편화된 패킷들을 식별하기 위한 고유한 번호를 나타낸다.
   * 플래그 필드는 단편화 금지(DF) 및 추가 단편화(MF) 여부르르 나타내는 정보를 포함한다.
   * 단편화 오프셋은 쪼개진 패킷이 원래 데이터에서 얼마나 떨어져 있는지를 나타낸다.
   * TTL은 패킷의 수명을 나타내며, 라우팅 과정에서 패킷이 무한정 네트워크에 남아있는 것을 방지한다.
   * 프로토콜 필드는 상위 계층 프로토콜을 나타내며, 캡슐화된 데이터의 종류를 명시한다.
   * 송신지 및 수신지 IP 주소 필드는 각각 패킷의 출발지와 목적지 IP 주소를 나타낸다.

 * IP 주소의 한계와 IPv6의 등장
   * IPv4 주소의 총 개수는 약 43억개로 부족하다. 이러한 문제를 해결하기 위해 IPv6가 등장했다.
   * IPv6는 128비트 주소를 사용하여 무한대의 IP 주소를 할당할 수 있다.

 * ARP(Address Resolution Protocol)의 필요성
   * 네트워크 통신 시 IP주소와 MAC주소가 함께 활용되지만, 때로는 통신하려는 호스트의 IP주소는 알아도 MAC 주소를 모르는 상황이 있다.
   * 올바른 통신을 위해서는 두 주소를 모두 알아야하므로, 이 문제를 해결하기 위해 ARP 프로토콜이 사용된다.
   * ARP는 IP주소를 통해 MAC 주소를 알아 내는 프로토콜이다.
   * ARP는 동일 네트워크 내 송수신 대상의 MAC 주소를 파악하는데 활용된다.
  
 * ARP 프로토콜의 동작 과정
   * ARP 프로토콜은 세 단계를 거쳐 동작한다.
     1. MAC주소를 알아내고자 하는 호스트느느 ARP 요청 메시지를 브로드 캐스트로 내보낸다.
     2. 요청을 받은 호스트 중 해당 IP 주소의 소유자는 자신의 MAC 주소를 담은 ARP 응답 메시지를 유니캐스트로 요청 호스트에게 전송한다.
     3. 이후 요청 호스트는 수신한 ARP 응답 메시지를 토대로 ARP 테이블이라는 정보를 갱신하여 IP 주소와 MAC 주소 매핑 정보를 기록한다.
    
 * ARP 패킷 형식과 ARP 테이블
    * ARP 요청은 응답과 ARP 패킷을 주고받는 과정이다.
    * ARP 패킷에는 오퍼레이션 코드(요청은1, 응답은 2), 송신지/수신지 하드웨어 주소(MAC주소), 송신지/수신지 프로토콜주소(IP주소)필드가 포함되어있다.
    * ARP 테이블은 IP주소와 매핑된 MAC주소를 기록하여 유지하는 특별한 정보이다.     

 * IP 주소의 기본 구조
    * IP 주소는 네트워크 주소와 호스트 주소, 크게 두 부분으로 나뉜다.
    * 네트워크 주소는 특정 네트워크를 식별하는데 사용된다.
    * 호스트 주소는 그 네트워크 내의 특정 호스트를 식별하는데 사용된다.
    * IP 주소는 총32비트로 구성된다.
    * IP 주소에서 네트워크 주소와 호스트 주소에 할당되는 비트 수는 항상 고정된 것은 아니며 네트워크의 규모에 따라 유동적으로 조절될 수 있다.

 * 클래스풀 주소 체계 : A, B, C 클래스
    * IP 주소의 클래스는 네트워크 크기에 따라 IP 주소 유형을 분류하는 기준이다.
    * A,B,C,D,E 총 5개의 클래스가 존재하지만, D와 E클래스는 주소 지정에 직접 사용되지 않는다.
    * A,B,C 클래스는 네트워크 주소를 표현하는 크기가 각 각 다르다.
    * A 클래스는 많은 호스트에 IP 주소를 할당할 수 있다
    * B 클래스는 네트워크 주소와 호스트 주소에 각각 2옥텟씩 할당된다.
    * C 클래스는 비교적 적은 수의 호스트에 적합하며 네트워크 주소를 3옥텟으로 표현한다.
  
 * 클래스별 IP 주소 범위 및 특징
    * A클래스의 IP 주소 범위는 0.0.0.0 부터 127.255.255.255 까지이며, 첫 번째 옥텟이 0으로 시작한다.
    * B클래스의 범위는 128.0.0.0 부터 191.255.255.255까지이며, 첫 두 비트는 10으로 시작한다.
    * C클래스의 범위는 192.0.0.0 부터 223.255.255.255까지이며, 처음 세 비트는 110으로 시작한다.
    * 클래스풀 어드레싱에서는 할당하고 싶은 호스트 수에 따라 적절한 클래스를 선택하는 것이 중요하다.
    * 호스트 주소가 전부 0인 IP 주소는 네트워크 주소로, 전부 1인 IP주소는 브로드캐스트 주소로 사용되기 때문에 할당할 수 없다.

  * 클래스풀 주소 체계의 한계
     * 클래스풀 주소 체계는 클래스별 네트워크 크기가 고정되어 있어 IP 주소 낭비가 발생하기 쉽다.
     * 클래스풀 주소 체계에서는 사전에 정해진 크기 외에 다른 크기의 네트워크를 구성하기 어렵다.  

  * 클래스리스 주소 체계의 등장
     * 클래스리스 주소 체계는 클래스 개념 없이 네트워크 영역을 나누고 IP주소를 할당하는 방식으로, 클래스풀 체계보다 유동적이고 정교한 네트워크 구성이 가능하다.
     * 클래스리스 주소 체계에서는 A,B,C 클래스의 구애 없이 네트워크 주소 비트를 유연하게 할당할 수 있다.
     * 네트워크 주소에 25비트 또는 20비트를 할당하는 등 정교한 네트워크 크기 조정이 가능하다.

  * Subnet 서브넷 마스크의 역할
     * 클래스리스 주소 체계에서는 클래스 개념을 사용하지 않기 때문에 네트워크 주소와 호스트 주소를 구분하는 수단이 필요하며, 이를 위해 서브넷 마스크를 사용한다.
     * 서브넷 마스크는 1과 0으로 이루어진 비트열로, 네트워크 주소는 1, 호스트 주소는 0으로 표현한다. 이를 통해 IP 주소 상에서 네트워크 주소와 호스트 주소를 구분할 수 있다.

  * 서브넷 마스크를 이용한 네트워크 주소 계산
     * IP 주소와 서브넷 마스크르르 비트 AND 연산하면 네트워크 주소 부분을 얻을 수 있다.
     * 호스트 주소 공간이 모두 0인 주소는 네트워크 자체를 의미한다.
     * 호스트 주소 공간이 모두 1인 주소는 브로드캐스트 주소를 의미한다.
     * 할당 가능한 호스트 IP 주소 범위는 네트워크 주소 + 1 부터 브로드캐스트 주소 -1 까지이다.
   
  * 표기법 사이다(CIDR)표기법
     * 서브넷 마스크를 표기할 때, 십진수 대신 IP 주소 상에서 서브넷 마스크의 1의 개수로 표현하는 사이더 표기법을 사용한다.

   * 공인 IP 주소 VS 사설 IP 주소
      * IP 주소는 크게 공인 IP 주소와 사설 IP 주소로 나눌 수 있다.
      * 공인 IP 주소는 전 세계에서 유일한 주소이며, 사설 IP 주소는 사설 네트워크 내에서만 유효한 주소이다.

   * 사설 네트워크와 사설 IP 주소
      * 사설 네트워크는 외부에 공개되지 않은, 우리끼리만 사용하는 네트워크를 의미한다.
      * 사설 IP 주소는 라우터에 의해 할당되며, 사설 네트워크 내에서만 유효하다.
      * 다른 사설 네트워크와 IP 주소가 중복될 수 있다.
    
   * NAT(Network Address Translation) : 사설 IP 주소를 공인 IP 주소로 변환하는 기술
      * NAT 라우터는 사설 네트워크에서 나가는 패킷의 출발지 IP 주소를 공인 IP 주소로 변경하고, 외부에서 들어오는 패킷의 목적지 IP 주소를 사설 IP 주소로 변경합니다.
    
   * IP 주소는 크게 정적 할당 방식과 동적 할당 방식으로 나뉜다.
     * 정적 IP 주소는 호스트에 직접 수작업으로 할당하는 방식을 의미한다.
     * 동적 IP 주소는 프로토콜을 기반으로 자동으로 할당받는 주소를 지정한다.
     * 정적 IP 주소를 부여할 때는 필수적인 값들들 입력해야한다.
        * 부여할 IP 주소 자체, 네트워크와 호스트를 구분하는 서브넷 마스크, 외부 네트워크로 나가는 첫 번째 경로인 게이트웨이주소, DNS 주소
      * 동적 할당은 DHCP(Dynamic Host Configuration Protocol)로 자동할당된다.
        * 공유기나 라우터가 DHCP 서버의 역할을 수행한다.
