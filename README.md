# 💻 컴퓨터 네트워크 공부 요약
공부한 네트워크 핵심 이론과 실습 내용을 기록하는 공간입니다.

------

## 🌐 네트워크 구성도 살펴보기
### 🏠 1. 홈 네트워크
* **구성 흐름 :** `인터넷` ➡️ `공유기` ➡️ `PC` 또는 `스마트폰`
> 💡 **공유기의 역할:** 인터넷(외부 네트워크)과 집 안의 기기들(내부 네트워크)을 서로 연결하고 데이터를 중중개해 주는 핵심 장비입니다.

---

### 🏢 2. 데이터 센터 네트워크 (3티어 아키텍처)
* **구성 흐름 :** `인터넷` ➡️ `방화벽` ➡️ `로드밸런서` ➡️ `WEB 서버` ➡️ `WAS 서버` ➡️ `DB 서버`
* **3티어 아키텍처란?**
  * 웹 서버(WEB), 애플리케이션 서버(WAS), 데이터베이스(DB)를 각각 독립된 계층으로 쪼갠 구조입니다.
  * **장점 :** 서버가 분리되어 있어 **보안**성이 높고, 트래픽 분산(**성능**) 및 관리(**유지보수**) 측면에서 매우 유리합니다.


## 🌐 크기에 따른 네트워크 분류 : LAN 과 WAN
* **LAN이란?** Local Area Network, 건물 안이나 특정 지역을 범위로 하는 네트워크.
* **WAN이란?** Wide Area Network, ISP가 제공하는 서비스를 사용하여 구축한 네트워크.

## 🌐 연결 형태에 따른 분류 : 네트워크 토폴로지
* 네트워크는 연결 형태에 따라 `스타형(성형)` 과 `메시형(망형)`으로 나눌 수 있다.
* 스타형 : 중앙 장비에 모든 장비가 연결된 형태, 장점 : 장애 찾기 쉽다, 관리가 쉽다. 단점 : 중앙 스위치 죽으면 전체 영향이간다. 
* 메시형 : 여러 노드들이 서로 전부 연결된 형태이며 일부 장비가 고장나도 다른 경로를 통해 통신이 가능 WAN에서 주로 사용되는 방식, 안전성 높지만 비용이 높다
* 버스형 : 하나의 선 공유 과거에 주요사용됨
* 링형 : 원형 연결, 일부 특수 환경에서 사용한다.
* 대부분의 네트워크는 LAN과 WAN이 혼합된 형태이다. **

## 🌐 네트워크 통신 방식 : 유니캐스트, 멀티캐스트, 브로드캐스트
* 유니캐스트 : 특정 대상과만 통신하는 방식 ex) 특정 사람과 통신, 1대1 통신, 카카오톡, SSH, 웹서핑 등
* 멀티캐스트 : 특정한 다수와 통신하는 방식 ex) 특정 그룹과 통신, IPTV
* 브로드캐스트 : 같은 네트워크 대역에 있는 모든 사용자와 통신하는 방식 ex) 모든 사람과 통신, ARP 요청
* 애니캐스트 : 가장 가까운 서버로 전달 ex) DNS 서버

## 🌐 프로토콜 : 네트워크 통신의 약속
* 프로토콜 : 컴퓨터가 네트워크 망에서 특정 컴퓨터와 통신할 때는 프로토콜이라는 것을 사용한다. 프로토콜은 일종의 약속 또는 양식이다.
* 프로토콜에는 이더넷, ICMP, IP, TCP, UDP 등 여러 종류가 있으며, 각각 사용하는 네트워크 주소가 다르다.
* IP : 주소지정
* TCP : 신뢰성
* UDP : 속도
* HTTP : 웹 통신
* HTTPS : 암호화 웹
* DNS : 이름 -> IP 변환
* ARP : IP -> MAC 변환
* ICMP : 네트워크 상태 확인
* SSH : 원격접속
* 흐름 : google.com 입력 -> DNS가 IP 찾음 -> TCP 연결 -> HTTP 요청 -> 서버가 응답하는 흐름

## 🌐 패킷의 구조와 역할
* 헤더, 페이로드, 트레일러 구조
* 패킷은 네트워크를 통해 주고받는 메시지의 기본 단위
* 헤더는 대부분 포함되지만, 트레일러는 없을 수도 있다.
* 헤더는 패킷의 송수신지 정보를 담고 있다.
* 1계층 bit
* 2계층 프레임
* 3계층 패킷
* 4계층 세그먼트

## 🌐 TCP/IP 모델 과 OSI 7 레이어 모델
* TCIP/IP 모델 : 네트워크 엑세스, 인터넷, 전송, 응용 4개 계층으로 이루어져있다.
* OSI 7레이어 모델은 ISO에서 지정한 모델로, 통신 전체를 체계적으로 다룬다.
* OSI 7계층 : 응용계층, 표현계층, 세션계층, 전송계층, 네트워크계층, 데이터링크계층, 물리계층 (응표세전네데물)
  * 1계층 물리계층 : 전기 신호 전달, 케이블, 허브
  * 2계층 데이터링크 : MAC 주소 기반 통신, 스위치, 같은 네트워크안에서 통신
  * 3계층 네트워크 : IP 주소 기반, 라우터, 다른 네트워크 찾기
  * 4계층 전송 : TCP/UDP 담당, PDU:세그먼트, 신뢰성/속도 관리
  * 5계층 세션 : 연결 관리
  * 6계층 표현 : 암호화/압축, SSL/TLS
  * 7계층 응용 : 사용자 프로그램, 브라우저, 메신저 / HTTP, FTP, SMTP
* 이더넷, IP, ICMP, ARP, TCP, UDP, HTTP 등의 프로토콜이 있다.
* TCP/IP가 더 실무적으로 되어있다.

## 🌐 인캡슐레이션/디캡슐레이션
* 패킷은 페이로드(데이터), 헤더(제어정보), 푸터(꼬리말) 로 구성되는데 대부분의 경우 헤더만 사용된다.
* 페이로드에 프로토콜을 헤더로 붙이는 과정을 캡슐화(Encapsulation)이라고 한다.
* 캡슐화는 상위 계층에서 하위 계층으로 내려가면서 프로토콜을 붙이는 과정이다.
* 패킷을 받는 쪽에서 디캡슐화(Decapsulation)과정을 통해 패킷을 확인한다.
* 하위 프로토콜부터 하나씩 벗겨내며 데이터를 확인하는 과정으로, 누가 누구에게 어떤 데이터를 보냈는지 순차적으로 파악한다
* 하위계층에서부터 상위 계층으로 올라가며 프로토콜 헤더를 제거하고 최종적으로 순수한 데이터만 얻게 된다.
* 인캡슐레이션 (응용데이터 -> TCP Header(포트 번호 포함)  -> IP Header(IP주소포함) -> Ethernet Header(MAC주소포함) -> 전송)
* 디캡슐레이션 (Ethernet제거 -> IP 제거 -> TCP 제거 -> 응용데이터 도착

## 🌐 네트워크 연결 부분
* 네트워크는 유선, 무선 연결방식으로 나뉨
* 유선 네트워크 대표는 이더넷(Ethernet)
  * `이더넷 Ethernet`
     * 현재 LAN의 표준 기술
     * MAC 주소를 사용
     * 스위치 기반
     * 프레임 단위 통신
     * CSMA/CD 기반(과거 허브 환경) : 먼저 통신 중인지 확인 하고 충돌 나면 기다렸다 재전송 요즘은 스위치 사용으로 충돌 거의 없음
* 반이중 /전이중
   * 반이중(Half Duplex) 통신 : 한 번에 한 방향만 통신한다. ex) 무전기
   * 전이중(Full Duplex) 통신 : 동시에 송수신 가능하다. ex) 전화

 ## 🌐 네트워크 회선
 * 회선이란? 데이터가 이동하는 통로 ex) 인터넷 회선, 광케이블, 전용선
 * 회선의 종류
    * 전용 회선 : 기업에서 많이 사용
    * 인터넷 회선 : 일반 인터넷, 여러 사용자가 공유
    * 인터넷 전용 회선 : 인터넷은 쓰는데 품질을 보장받는 회선, 기업에서 주로 사용
    * VPN : Virtual Private Network 인터넷 위에 안전한 전용망처럼 사용하는 기술
    * 광케이블 : 빛으로 데이터를 전달, 빠르고, 노이즈적고, 장거리 가능하다.
  
 ## 🌐 네트워크 구성 요소
 * NIC : Network Interface Card 컴퓨터를 네트워크에 연결하는 역할
 * 케이블과 커넥터 : CAT5e (1G), CAT6(10G 일부), CAT6A(10G)
 * 허브 : 1계층 장비, 들어온 데이터를 모든 포트로 전달한다. 충돌 많고, 보안에 취약
 * 스위치 : 2계층 장비, MAC 주소 기반 전달한다. MAC Address Table이 있다
 * 라우터 : 3계층 장비, 다른 네트워크와 연결, 라우팅 테이블
 * 로드밸런서 : 트래픽 분산 장비 : 서버 여러 대로 분산, 부하 분산, 장애 대응, 성능 향상
 * 보안장비 : 방화벽 (허용과차단), IPS(이상 트래픽 탐지/차단)
 * IDS/ IPS : IDS 침입탐지, 알림중심 / IPS : 침입차단, 자동 대능 가능
 * VPN : 가상 사설망, 인터넷을 통해 안전하게 내부망 접속
 * NAT : 사설IP <-> 공인 IP 변환
 * DHCP : IP 자동 할당

  ## 🌐 MAC 주소
  * 랜카드(NIC)에 부여된 물리 주소
  * 2계층 주소
  * OO:1A:2B:3C:4D:5E / 48비트 사용
  * 전 세계 유일하며, 제조사 정보 포함되어있고, 변경 가능은 하지만 기본은 고유값이다.
  * MAC 주소는 앞 24비트는 제조사 뒤 24비트는 장비 고윳값이다.

 ## 🌐 IP 주소
  * 네트워크에서 사용하는 논리 주소
  * IPv4 형태, 192.168.0.1 32비트
  * 다른 네트워크 찾기 위해 필요하다.
  * 라우터가 IP 기반 동작한다.
  * 공인IP, 사설IP
     * 공인IP : 인터넷에서 유일한 주소(통신사 제공)
     * 사설IP : 내부망용 주소 / 192.168.x.x , 10.x.x.x, 172.16.x.x
     * 사설 IP는 IPv4 부족때문에 사용한다.
  * NAT : 사설 IP <-> 공인 IP 변환
  * 서브넷 마스크 : 네트워크/호스트 구분용도
  * CIDR 표기 : 앞 24비트가 네트워크 부분이다.
  * 기본 게이트웨이 : 외부로 나갈 출구
  * DNS : 도메인 -> IP 변환 ex) goolgle.com -> 142.x.x.x
  * IP 통신 과정 : 내 pc -> 게이트웨이 -> 라우터 -> 인터넷 -> 목적지 서버
  * TTL : Time To Live , 패킷 생존 시간, 라우터 지날 때 감소하며 0 되면 폐기, 루프 방지 목적이다.

 ## 🌐 TCP와 UDP
  * TCP 는 신뢰성 중심, 연결형, 순서보장, 재전송, 흐름제어 , 혼잡제어
  * TCPsms 3-Way Handshaek 로 연결
     * SYN, SYN-ACK, ACK
  * TCP는 안정적이고 데이터 손실 복구가 되나 느리다. 주로 웹, 로그인, 파일전송 , DB에서 사용한다.
  * UDP는 속도 중심이고 비연결형, 빠르다, 신뢰성없다, 순서보장 하지 않는다.
  * 주로 게임, 스트리밍 에서 사용한다.

  ## 🌐 이더넷의 이해와 중요성
   * 이더넷은 현대 LAN 환경, 특히 유선 LAN에서 가장 널리 사용되는 기술이다.
   * 물리 계층에서의 통신 매체 간 정보 교환 규칙과 데이터 링크 계층에서 두 호스트 간 메시지 교환 형식을 정의하는 핵심 기술이다.
   * 오늘날 대부분의 유선 LAN 환경은 이더넷 기술을 기반으로 구성된다.
   * 물리 계층의 케이블과 데이터 링크 계층에서 주고받는 프레임이 모두 이더넷 규격을 따른다.

  ## 🌐 이더넷 프레임
   * 데이터 링크 계층에서 메시지를 주고받는 단위는 `프레임`이라고 한다.
   * 이더넷 네트워크에서 사용되는 프레임을 이더넷 프레임 이라 한다.
   * 이더넷 프레임은 헤더, 페이로드(데이터), 트레일러로 구성된다.
   * 이더넷 프레임의 헤더는 크게 프리앰블, 수신지 MAC 주소(6바이트), 송신지 MAC 주소(6바이트), 타입(2바이트), 길이 필드로 구성된다.
   * 프리앰블은 8바이트 길의 비트열로, 수신지에서 프레임 수신을 준비하도록 동기화하는 역할을 한다.
   * MAC 주소는 수신지 및 송신지 MAC 주소 필드에 명시되는 네트워크 인터페이스마다 부여되는 6바이트(48비트) 길이의 물리적 주소이다.

  ## 🌐 NIC(네트워크 인터페이스 컨트롤러)란?
   * NIC는 호스트와 통신 매체를 연결하는 인터페이스 역할을 수행하는 장비이며 흔히 네트워크 카드, 랜 카드 등으로 불린다.
   * NIC는 통신 매체의 전기신호를 호스트가 이해할 수 있는 정보로 변환하거나, 호스트의 디지털 정보를 통신 매체 신호로 변환하는 역할을 한다.
   * 호스트가 네트워크를 통해 송수신하는 모든 정보는 NIC를 거치게 된다.
   * NIC는 MAC 주소를 인식하여 수신한 프레임의 송수신지 MAC 주소를 확인하고, 자신과 관련 없는 프레임을 폐기하거나 일치하는 프레임을 역캡슐화 수행한다.

  ## 🌐 물리 계층과 데이터 링크 계층 장비의 핵심 차이
   * 물리 계층의 네트워크 장비는 주소 개념이 없어 송수신되는 정보에 대한 조작이나 판단을 할 수 없다.
   * 데이터 링크 계층 이상의 장비는 MAC 주소와 같은 주소 개념을 가지고 있어 송수신 정보를 특정하고 판단하여 조작할 수 있다.
  
  ## 🌐 물리계층 대표 장비 `허브` (1계층)
   * 허브는 `리피터 허브` 또는 `이더넷 허브` 등으로 불리며, 여러 호스트를 연결하는데 사용된다.
   * 허브의 첫 번째 전송 특징은 모든 포트로 해당 메시지를 내보낸다. 이러한 것은 불필요한 트래픽을 과도하게 증가시키는 문제점을 야기시킨다.
   * 허브의 두 번재 특징은 반이중 방식으로 통신한다. ex) 무전기
   * 모든 포트로 신호를 보내고 반이중 방식으로 통신하는 특성 때문에 충돌(Collision)문제가 발생한다. 충돌이 발생할 수 있는 영역을 콜리전 도메인 이라고 한다.
   * 충돌을 줄이려고 CSMA/CD 프로토콜을 이용한다.

  ## 🌐 데이터링크계층 대표 장비 `스위치` (2계층)
   * 스위치는 허브와 달리 어떤 MAC 주소를 가진 호스트가 어떤 포트에 연결되어 있는지 식별할 수 있다.
   * 스위치는 전 이중 모드 통신을 지원하여 근본적으로 CSMA/CD 프로토콜이 필요가 없다.
   * 일반적으로 스위치라하면 L2 스위치를 말하긴한다.
   
   * 스위치의 주요기능으로는 첫 번째는 `MAC 주소 학습 기능`이다.
      * 이 기능은 어떤 포트에 어떤 MAC 주소를 가진 호스트가 연결되어 있는지 학습하여 기억한다.
      * 스위치가 전달받은 신호를 원하는 목적지 MAC 주소가 연결된 포트로만 내보낼 수 있게 해준다.
      * MAC 주소 학습을 위해 스위치는 MAC 주소 테이블이라는 특별한 정보를 저장한다.
      * 이 테이블은 스위치 포트와 연결된 호스트의 MAC 주소 간의 연관 관계를 나타내는 표와 같으며, 스위치는 이 테이블을 통해 특정 포트에 연결된 호스트의 MAC 주소를 알 수 있다.
      * MAC 주소 테이블은 플러딩, 포워딩, 필터링, 에이징 을 통해 생성되고 관리된다.
      * `스위치 플러딩`은 처음에 MAC 주소 테이블이 비어있어 호스트의 위치를 모르면 허브처럼 모든 포트로 프레임을 전송한다.
      * MAC 주소를 학습한 이후에는 A->C로 전송할 때 다른 포트로 보내지 않는다 이것을 `필터링`이라고한다.
      * 연결된 호스트에 포트로만 프레임을 내보내는것을 `포워딩`이라고 한다.
      * `에이징`은 MAC 주소 테이블에 있는 항목을 일정 시간이 지나면 삭제하는 기능이다.
    
   * 스위치의 두 번째 주요 기능은 `VLAN(Virtual LAN)` 이다.
      * VLAN은 논리적으로 나누어진 가상의 LAN을 의미하며, 한 대의 스위치로 여러 개의 가상 LAN을 만들 수 있게 해준다.
      * VLAN을 이용하면 불필요한 트래픽을 분배하여 성능 저하를 방지할 수 있다.
      * VLAN을 사용하면 물리적인 스위치가 한 대라도 여러 대의 스위치가 있는 것처럼 호스트의 물리적 위치와 관계없이 논리적인 단위로 LAN을 구성하고 나눌 수 있다.
    
     ## 🌐 STP (Spanning Tree Protocol)
     
     ### 1. STP의 개념 및 필요성
     * **정의:** 가상이나 물리적인 네트워크 루프(Loop)를 방지하기 위해, 특정 포트를 논리적으로 차단하는 **루프 방지 프로토콜**입니다.
     * **실무 활용:** 네트워크의 **이중화(Redundancy)**를 구현하여 특정 장비나 회선에 장애가 발생해도 서비스가 중단되지 않도록 **네트워크 안정성**을 확보합니다.
     
     ---
     
     ### 🔄 2. STP 기본 동작 개념
     스위치 간의 협상을 통해 다음과 같은 순서로 루프를 제거하고 최적의 경로를 계산합니다.
     
     1. **Root Bridge 선출:** 전체 네트워크의 중심이 될 기준 스위치(중앙 스위치)를 선출합니다.
     2. **최적 경로 계산:** Root Bridge를 기준으로 각 스위치까지의 가장 가깝고 효율적인 경로를 계산합니다.
     3. **차단 포트 결정 (루프 제거):** 최적 경로 외에 루프를 유발할 수 있는 나머지 우회 경로는 포트를 차단(Blocking)하여 트래픽을 막습니다.

---

### 🚀 3. 향상된 STP 종류

실무 환경과 네트워크 규모에 따라 기존 STP의 단점을 보완한 프로토콜을 사용합니다.

| 프로토콜명 | 풀네임 (Full Name) | 주요 특징 및 실무 활용 |
| :--- | :--- | :--- |
| **RSTP** | Rapid Spanning Tree Protocol | • **빠른 복구 속도** (기존 STP의 30~50초 걸리던 수렴 시간을 2~3초 내외로 단축)<br>• 현대 실무 네트워크에서 가장 기본적으로 많이 사용됨 |
| **MST** | Multiple Spanning Tree Protocol | • **여러 개의 VLAN을 그룹으로 묶어** STP 인스턴스를 관리<br>• 스위치의 CPU 자원을 아낄 수 있어 **대규모 환경**에 필수적임 |
## 🌐 네트워크 계층 (3계층)

### 1. 네트워크 계층과 IP 프로토콜
* **네트워크 계층의 핵심:** 물리 계층과 데이터 링크 계층의 한계를 극복하고, **IP 주소 지정**과 **라우팅**을 가능하게 하는 네트워크 기능의 핵심입니다.
* **IP 프로토콜 ($IPv4$ vs $IPv6$):** 가장 중추적인 역할을 수행하며, 특별한 언급이 없다면 기본적으로 `IPv4`를 의미합니다.
  * **IPv4:** 32비트 구조 (약 43억 개 주소 제한으로 고갈 문제 발생)
  * **IPv6:** 128비트 구조 (IPv4의 한계를 극복하기 위해 등장, 무한대에 가까운 주소 할당 가능)

---

### 📦 2. IP 프로토콜의 핵심 기능

#### 🔹 주소 지정 (Addressing)
* IP 주소 체계를 통해 호스트가 속한 **네트워크 주소**와 **호스트 주소** 자체를 식별하는 기능입니다.
* IPv4 주소는 **32비트**로 구성되며, 4개의 **옥텟(Octet)**으로 구분되어 각 옥텟은 `0~255`까지의 숫자로 표현됩니다.

#### 🔹 단편화 (Fragmentation)
* 전송하려는 패킷 크기가 **MTU(Maximum Transmission Unit, 최대 전송 단위)**보다 클 때 발생합니다.
* **MTU 사양:** 일반적으로 `1500바이트` 크기를 가집니다.
* **작동 방식:** 큰 데이터를 MTU 이하의 작은 패킷 조각으로 쪼개어 전송하며, 수신 측에서 이 패킷들을 다시 하나로 재조합합니다.

#### 📄 IPv4 패킷 헤더의 주요 필드
| 필드명 | 역할 설명 |
| :--- | :--- |
| **식별자 (Identification)** | 단편화된 패킷들이 원래 하나의 데이터였음을 식별하기 위한 고유 번호 |
| **플래그 (Flags)** | 단편화 금지(DF) 및 추가 단편화 존재 여부(MF)를 나타내는 정보 |
| **단편화 오프셋 (Fragment Offset)** | 쪼개진 패킷 조각이 원래 데이터에서 얼마나 떨어져 있는지의 위치 값 |
| **TTL (Time To Live)** | 패킷의 수명. 라우팅 중 패킷이 네트워크 내에서 무한 루프를 도는 것을 방지 |
| **프로토콜 (Protocol)** | 상위 계층 프로토콜을 명시하여 캡슐화된 데이터의 종류를 안내 |
| **송신지 / 수신지 IP** | 패킷의 출발지 IP 주소와 최종 목적지 IP 주소 |

---

### 🔍 3. ARP (Address Resolution Protocol)

* **필요성:** 네트워크 통신 시 IP 주소는 알고 있지만, 데이터 링크 계층 전송에 필수적인 **MAC 주소를 모르는 상황**을 해결하기 위해 사용됩니다. (동일 네트워크 내에서 작동)

#### 🔄 ARP 프로토콜 동작 3단계
1. **ARP 요청 (Broadcast):** MAC 주소를 알아내기 위해 네트워크 전체에 브로드캐스트 메시지를 전송합니다.
2. **ARP 응답 (Unicast):** 해당 IP를 가진 호스트가 자신의 MAC 주소를 담아 요청 호스트에게 유니캐스트로 답변을 보냅니다.
3. **ARP 테이블 갱신:** 응답을 받은 호스트는 IP와 MAC 주소 매핑 정보를 `ARP 테이블`에 기록하고 유지합니다.

> 💡 **ARP 패킷 구성:** 오퍼레이션 코드(요청: 1, 응답: 2), 송신지/수신지 하드웨어 주소(MAC), 송신지/수신지 프로토콜 주소(IP)

---

### 🔢 4. IP 주소 체계의 발전 (Classful vs Classless)

#### 🅰️ 클래스풀 주소 체계 (Classful Addressing)
네트워크 크기에 따라 사전에 고정된 구분을 사용하는 방식입니다. (D, E 클래스는 특수 목적용)
* **A 클래스:** `0.0.0.0` ~ `127.255.255.255` (첫 비트 `0` / 대규모 네트워크)
* **B 클래스:** `128.0.0.0` ~ `191.255.255.255` (첫 두 비트 `10` / 중규모 네트워크)
* **C 클래스:** `192.0.0.0` ~ `223.255.255.255` (첫 세 비트 `110` / 소규모 네트워크)
* **⚠️ 치명적 한계:** 정해진 크기로만 쪼갤 수 있어 IP 주소의 낭비가 매우 심합니다.

#### 🅱️ 클래스리스 주소 체계 (Classless Addressing)
클래스 구애 없이 네트워크 주소 비트를 유연하게 조정하는 방식입니다. (예: 20비트, 25비트 조정 가능)
* **서브넷 마스크 (Subnet Mask):** 클래스가 없기 때문에 네트워크와 호스트 영역을 구별하기 위해 도입되었습니다. (`1`은 네트워크 영역, `0`은 호스트 영역)
* **CIDR (사이더) 표기법:** 서브넷 마스크의 `1`의 개수를 IP 주소 뒤에 직관적으로 표현하는 방식입니다. (예: `/24`)

#### 🧮 네트워크 주소 계산법
* **네트워크 자체 주소:** IP 주소와 서브넷 마스크를 비트 **AND 연산**한 결과 (호스트 주소가 모두 `0`인 상태)
* **브로드캐스트 주소:** 호스트 주소 공간이 모두 `1`인 상태
* **실제 할당 가능한 호스트 IP 범위:** `네트워크 주소 + 1` ~ `브로드캐스트 주소 - 1`

---

### 🛡️ 5. 공인 IP vs 사설 IP & NAT

* **공인 IP 주소 (Public IP):** 전 세계에서 유일하며 인터넷 망에서 직접 통신 가능한 주소입니다.
* **사설 IP 주소 (Private IP):** 사설 네트워크(외부 미공개 내부망) 내부에서만 유효한 주소입니다. 라우터가 할당하며 다른 사설망과 주소가 중복될 수 있습니다.

#### 🔄 NAT (Network Address Translation) 기술
* **정의:** 사설 IP 주소를 공인 IP 주소로 변환하여 외부 인터넷과 통신할 수 있게 돕는 기술입니다.
* **동작:** NAT 라우터가 내부에서 나가는 패킷의 출발지 IP를 공인 IP로 변환하고, 외부에서 들어오는 패킷의 목적지 IP를 사설 IP로 맵핑하여 돌려줍니다.

---

### ⚙️ 6. IP 주소 할당 방식

#### 📍 정적 할당 (Static IP)
* 사용자가 호스트에 수작업으로 고정 IP를 직접 입력하는 방식입니다.
* **필수 입력 값 4가지:** `부여할 IP 주소`, `서브넷 마스크`, `게이트웨이 주소(외부 네트워크 경로)`, `DNS 주소`

#### ⚡ 동적 할당 (Dynamic IP)
* **DHCP (Dynamic Host Configuration Protocol)** 프로토콜을 기반으로 네트워크 접속 시 자동으로 주소를 할당받는 방식입니다.
* 일반적으로 가정이나 사무실의 공유기, 혹은 라우터가 **DHCP 서버** 역할을 수행하여 IP를 나누어 줍니다.
